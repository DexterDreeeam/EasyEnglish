# Agent Module — Interface

> **Status:** **draft (proposal, iter-016).** Frozen at the end of iter-016.
> Once frozen, breaking changes to anything in `## Public` require an ADR.

Crate name: `ee-agent`.

## Public

### `Agent`

```rust
pub struct Agent { /* opaque */ }

impl Agent {
    /// Build with a fully populated registry and the default Instructions
    /// used when callers pass `None` overrides. The registry is owned;
    /// callers that need to swap the registry should rebuild the Agent.
    pub fn new(
        registry:             ee_skill::SkillRegistry,
        default_instructions: Instructions,
    ) -> Self;

    /// Orchestrate skills for `word`. `hit` is the result from
    /// `ee-core::AppState`, may be `None` (the dict didn't know the word).
    /// `overrides` shadow `default_instructions` for this call only and do
    /// NOT mutate Agent state.
    ///
    /// Per-skill failure is reported via `AgentResult.errors`, not as an
    /// `AgentError`. See `Agent/.design.md` §4 for the policy.
    pub fn run(
        &self,
        word:      &str,
        hit:       Option<&ee_core::LookupHit>,
        overrides: Option<Instructions>,
    ) -> Result<AgentResult, AgentError>;

    pub fn default_instructions(&self) -> &Instructions;
    pub fn set_default_instructions(&mut self, instr: Instructions);

    /// Borrow the registry. Useful for UI's "skills picker".
    pub fn registry(&self) -> &ee_skill::SkillRegistry;
}
```

### `Instructions`

```rust
#[derive(Clone, serde::Serialize, serde::Deserialize)]
pub struct Instructions {
    pub level:             CefrLevel,
    pub region:            Region,
    pub formality:         Formality,
    /// Honoured by skills whose output cardinality is configurable
    /// (e.g. `examples.N`). Hard-capped at 10 by deserialisation.
    pub max_examples:      u8,
    /// Doubles as run order. Empty → `Agent::run` returns
    /// `AgentError::NoSkillsEnabled`.
    pub enabled_skill_ids: Vec<String>,
    /// 0 disables the budget; > 0 enforces it as documented in
    /// `Agent/.design.md` §5 (current skill is not preempted).
    pub latency_budget_ms: u32,
    /// Free-form plugin configuration. Stable serialisation order (BTreeMap)
    /// so the same Instructions diff-stably in `product.json`.
    pub custom:            std::collections::BTreeMap<String, String>,
}

impl Default for Instructions {
    /// CEFR B2 / Region US / Formality Neutral / 3 examples /
    /// `["ipa.american", "examples.3", "formality.note"]` /
    /// 1500ms budget / empty custom.
    fn default() -> Self;
}
```

### Re-exported preference enums

These types are defined in `ee-skill` (so Skill is self-contained) and
re-exported here so Agent's callers don't need to import Skill:

```rust
pub use ee_skill::{CefrLevel, Region, Formality};
```

### Result

```rust
#[derive(Clone, serde::Serialize, serde::Deserialize)]
pub struct AgentResult {
    pub word:         String,
    pub base:         Option<ee_core::LookupHit>,
    /// In chronological run order — see `Agent/.design.md` §9 q.4.
    pub annotations:  Vec<ee_skill::Annotation>,
    /// Ids of skills whose `run` returned `Ok`. Same length as
    /// `annotations`, same order.
    pub used_skills:  Vec<String>,
    /// Skills that returned a non-`NotApplicable` error.
    pub errors:       Vec<(String, ee_skill::SkillError)>,
    /// Wall-time of the whole orchestration, in milliseconds.
    pub took_ms:      u32,
}
```

### Errors (pipeline-level only)

```rust
#[derive(thiserror::Error, Debug)]
pub enum AgentError {
    #[error("word is empty after trimming")]
    EmptyWord,

    #[error("instructions.enabled_skill_ids is empty")]
    NoSkillsEnabled,

    #[error("instructions reference unknown skill id: '{0}'")]
    UnknownSkillId(String),
}
```

> Per-skill failures are NOT `AgentError`. They land in `AgentResult.errors`.

## Private (not re-exported)

- `src/runner.rs` — the execution loop + budget tracker.
- `src/defaults.rs` — `Instructions::default()` constants; reachable only
  via `Default::default`.
- A `tracing::instrument` wrapper on `Agent::run` lives in `src/agent.rs`
  but is not part of the surface; callers should not assume any specific
  span shape.

## Dependencies

Workspace deps:

- `ee-core` (for `LookupHit`)
- `ee-skill` (for `Skill`, `Annotation`, `SkillError`, `SkillInput`, `SkillContext`, the 3 enums)
- `serde` + `serde_json`
- `thiserror`
- `tracing` (info per skill call, warn on `SkillError::Internal`)
- *(no rusqlite, no httplib, no async runtime)*

## Forbidden

- `pub fn` exposing the internal `Runner`.
- `async fn` anywhere in the public surface (iter-016 scope).
- Mutating the registry after `Agent::new`.
- Skill discovery via `inventory!` / linker-magic (must come through the
  registry passed at construction).
