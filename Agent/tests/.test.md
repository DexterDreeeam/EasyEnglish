# Agent Module — Test Specification (iter-016)

Tests will live in `Agent/tests/` once iter-016 ships. This file is the
contract. Every test below maps to one `#[test]` fn; every `#[test]` fn in
the eventual code maps back here.

Tests use:

- An empty `SkillRegistry` for "no skills" cases.
- A handful of in-test `FakeSkill`s (reused from `ee-skill`'s test
  fixtures via `dev-dependencies = { ee-skill = { path = "../Skill",
  features = ["test-fakes"] } }`).
- A mock clock for budget tests so wall-time flake never happens.

## Test files

| File | Covers |
|---|---|
| `test_instructions.rs` | `Instructions::default()` shape + serde + `max_examples` cap + BTreeMap ordering |
| `test_agent_basic.rs` | Construction, accessors, pipeline-level `AgentError` cases |
| `test_agent_orchestration.rs` | Skill running, ordering, partial failure, `NotApplicable` filtering |
| `test_agent_budget.rs` | Latency budget on / off / hit-mid-pipeline (uses mock clock) |
| `test_agent_with_core.rs` | Integration: real `ee-core::LookupHit` round-trip + `AgentResult` serde |

## `test_instructions.rs`

- `default_instructions_match_documentation` — every field of `Instructions::default()` matches the literal values stated in `Agent/.interface.md`.
- `default_enabled_skill_ids_exact` — exactly `["ipa.american", "examples.3", "formality.note"]` in that order.
- `serialize_deserialize_round_trip_json`.
- `max_examples_above_ten_is_capped_to_ten_on_deserialize`.
- `max_examples_zero_round_trips_zero` (zero means "skill picks").
- `enabled_skill_ids_preserves_order_through_serde` — `["c", "a", "b"]` survives as `["c", "a", "b"]`.
- `custom_btreemap_serializes_in_sorted_key_order` — guards diff stability in `product.json`.
- `instructions_is_clone`.

## `test_agent_basic.rs`

- `new_stores_default_instructions` — `agent.default_instructions() == passed`.
- `set_default_instructions_replaces_them`.
- `registry_accessor_returns_passed_registry_len`.
- `run_empty_word_returns_empty_word_error` — `""`, `"   "`, `"\t\n"` all → `Err(EmptyWord)`.
- `run_whitespace_word_is_trimmed_before_check` — `"  hello  "` does NOT raise `EmptyWord`.
- `run_no_enabled_skills_returns_no_skills_enabled` — Instructions with `enabled_skill_ids: []` → `Err(NoSkillsEnabled)`.
- `run_unknown_skill_id_returns_unknown_skill_id` — Instructions referencing `"nope.nope"` not in registry → `Err(UnknownSkillId("nope.nope".into()))`.
- `run_unknown_skill_id_short_circuits_before_any_skill_runs` — uses a FakeSkill spy to assert it was never called when an earlier id is unknown.

## `test_agent_orchestration.rs`

For all tests, registry contains 3 FakeSkills configurable per scenario:
`a` returns `Annotation`, `b` returns `NotApplicable`, `c` returns
`DataSourceUnavailable("offline")`.

- `single_skill_records_annotation_and_used_id` — enabled `["a"]` → 1 annotation, `used_skills == ["a"]`, `errors` empty.
- `multiple_skills_run_in_enabled_order` — enabled `["c", "a", "b"]` (note `b` is silent-skip) → `annotations.len() == 1` (from `a`), `used_skills == ["a"]`, `errors == [("c", DataSourceUnavailable(...))]`.
- `not_applicable_is_silently_dropped` — `b` alone → empty annotations, empty errors, empty used_skills.
- `data_source_unavailable_lands_in_errors_vec` — `c` alone → empty annotations, errors has `("c", _)`.
- `internal_error_lands_in_errors_vec` — FakeSkill returning `Internal("boom")` → in errors.
- `partial_failure_other_skills_still_run` — enabled `["c", "a"]` → both errors and annotations populated.
- `overrides_do_not_mutate_default_instructions` — `agent.run(_, _, Some(custom))`, then `agent.default_instructions()` is unchanged byte-for-byte.
- `each_skill_receives_input_word_unchanged` — FakeSkill spy records its inputs; assert `word` and `hit` and `ctx` match what was passed.
- `each_skill_receives_skill_context_built_from_instructions` — FakeSkill spy → `ctx.region == instructions.region`, etc.
- `used_skills_is_chronological_not_enabled_order` — enabled `["c", "a"]`, `c` fails, `a` succeeds → `used_skills == ["a"]` (only `a`).
- `took_ms_is_populated_and_nonnegative`.

## `test_agent_budget.rs`

Uses a `MockClock` fixture (constructor `Agent::with_clock(...)` is a
`#[doc(hidden)] #[cfg(test)]` ctor, NOT part of the public surface).

- `budget_zero_runs_all_skills_regardless_of_elapsed_time`.
- `budget_positive_runs_all_skills_when_under_budget`.
- `budget_exceeded_after_first_skill_cancels_remainder` — 2 skills queued, clock advances past budget after the first, second is recorded as `("b", SkillError::Cancelled)` and never invoked.
- `current_skill_is_not_preempted` — first skill takes 9999ms (mocked), runs to completion even though budget was 100ms. Documents the "+1 skill duration" tolerance from `Agent/.design.md` §5.
- `cancelled_skills_are_in_errors_not_used_skills`.

## `test_agent_with_core.rs`

- `run_passes_lookup_hit_some_to_skill_input` — set up a `ee-core::DictStore`, perform a lookup, pass the resulting `LookupHit` to Agent → FakeSkill spy sees `Some(&hit)`.
- `run_handles_lookup_hit_none_and_passes_through_to_skill_input` — FakeSkill spy sees `None`.
- `agent_result_serialises_for_ui` — round-trip an `AgentResult` containing a `LookupHit::Dict(Entry)` through `serde_json` without loss.
- `agent_result_serialises_with_errors_present` — errors vector round-trips, including the `SkillError` variants.
- `agent_run_with_default_config_agent_block_uses_documented_defaults` — load `product.json` lacking the `agent` block via `ee-core::Config::agent_instructions()` → equals `Instructions::default()`.
