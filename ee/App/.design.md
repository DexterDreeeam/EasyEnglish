# App Module — Design Specification & Global UX Constitution

`App/` is the **platform-integration umbrella** for EasyEnglish. It is a directory grouping holding three sibling crates, one per supported desktop OS:

| Crate | Crate name | Target OS | Primary Platform Integrations |
|---|---|---|---|
| [`Win`](Win/.design.md) | `ee-win` | Windows | Tray icon, native low-level mouse hooks, Win32 message-only window |
| [`Mac`](Mac/.design.md) | `ee-mac` | macOS | Status bar item, Cocoa global event monitors, NSWindow overlays |
| [`Linux`](Linux/.design.md) | `ee-linux` | Linux | AppIndicator/StatusNotifierItem tray, X11/Wayland input overlays |

All platform crates are bound to a unified, shared **Interactive UX Constitution** to guarantee identical user behavior across all desktop environments.

---

## 1. Global Interactive UX Specification

To maintain a consistent, distraction-free, and highly responsive user experience across Windows, macOS, and Linux, every platform shell implementation MUST conform strictly to the following architectural design:

### 1.1 Pure Bordered Input Box
- **Compact Floating HUD**: There is no window decoration, title bar, close/minimize buttons, or outer canvas background. Only the search box itself is rendered as a standalone floating container.
- **Search Bar Layout**: An elegant, solid deep black (`#0F0F0F`) rectangular block with slight rounding (`rounding(4.0)`), enclosed by a crisp gray border (`stroke(1.5)`), sized strictly to a compact width of `550px` (extended 20% based on feedback).
- **No Magnifying Glass**: Clean input interface without magnifying glass icons, search buttons, or indicators on the right. Typing is completely silent and distraction-free.

### 1.2 System-Tray Resident Daemon
- **Silent Background Startup**: On execution, the application starts completely hidden with no main window.
- **Persistent Tray Icon**: Registers an OS-native status indicator in the system tray / menu bar.
- **Tray Menu**: Right-clicking the tray icon reveals a native popup menu containing:
  - **`Show Flyout`**: Wakes up and focuses the floating search box.
  - **`Exit`**: Stops all background loops, deletes status icons, uninstalls hooks, and cleanly terminates the process.

### 1.3 Global Hot-Trigger
- **Universal Shortcut**: Pressing **`LeftAlt + LeftShift + MouseWheelUp`** or **`LeftAlt + LeftShift + UpArrow`** globally instantly triggers the search overlay.
- **Platform Hook**: The system-wide low-level mouse hook intercepts scroll-up events, checks key modifier states, and forces the hidden window to become visible and receive active keyboard focus.
- **Bypass Event Loop Suspension**: Each platform MUST implement native window showing commands (such as Win32 `ShowWindow` or Cocoa `makeKeyAndOrderFront`) directly from the hook thread to wake up suspended window event loops instantly.

### 1.4 Focus Grace Period & Auto-Close
- **Escape to Hide**: Pressing the `Escape` key immediately clears the text input buffer, clears search results, and hides the search window back into the tray.
- **Hide on Focus Loss**: Clicking anywhere outside the search window (losing foreground OS focus) immediately clears state and hides the window back into the tray.
- **DPI/OS Animation Grace Period**: Because window show animations take time (making the window temporarily unfocused during first frames), the app enforces a **15-frame grace period** on wakeup. Loss-of-focus check is active only after the grace frames elapse, preventing immediate self-closing on wakeup.

### 1.5 High-DPI Aware Vertical Center Auto-Resizing
- **Dynamic Sizing**: The window's height dynamically adapts in real-time based on content (starting at `56px` for a plain input box, and scaling up to `356px` to wrap query results).
- **DPI Scaling Alignment**: System metrics retrieved from the OS are scaled down using `ctx.pixels_per_point()` to align with egui's logical point coordinate system.
- **Top-Locked Stable Centering**: The window's absolute position on the screen is dynamically positioned using the constant base height:
  - `x = (logical_screen_width - 550.0) / 2.0`
  - `y = (logical_screen_height - 56.0) / 2.0`
  - This locks the top edge of the window in place. As the window's physical height expands downwards, the input box remains **100% stationary**, and the results grow downwards smoothly.

### 1.6 Card & Card Preview Layout
- **Exact Match (Card)**: There is **at most one Card**. When the query has an exact vocabulary match, it is rendered as a full, rich **Card** (pronunciation IPA, definitions, inflections, and bilingual examples). When there is no exact match, no Card is shown.
- **Multiple exact matches by priority**: Providers are consulted in priority order (Note > Offline Dict > Search). The highest-priority exact match becomes the single Card; any **lower-priority exact matches are placed at the top of the Card Previews**, above the fuzzy/prefix suggestions.
- **Fuzzy/Prefix Suggestions (Card Preview)**: At most **5** Card Previews are shown — lower-priority exact matches first, then fuzzy/prefix candidates — as compact, single-line rows displaying the word and its `major` meaning (summarized from primary meanings).
- **Always-present "Search on Bing"**: The **bottom row** of the results pane is always a `Search on Bing: <word>` entry whenever the input is non-empty — in every state, including when there are zero dictionary results. It is hidden only when the input is empty. Activating it (mouse click, or `Enter`/`Space` while focused) opens that word on Bing.
- **Instant Search with Cancellation**: Search is triggered instantaneously on typing. Modifying the query immediately cancels previous background query threads to preserve CPU/SQLite resources.
- **Flicker-Free Double Buffering**: To prevent layouts from shaking and blinking as records load one-by-one on every keystroke, the UI uses a double buffer. The displayed list is only updated on the screen when the complete async query terminates cleanly.

### 1.7 Keyboard Focus Navigation
- **Arrow-Key and Scroll Focus**: Pressing **Up/Down Arrow keys** or scrolling shifts focus through the text input bar (`index 0`), the exact Card (`index 1`, when present), the list of Card Previews, and finally the always-present "Search on Bing" entry (the last index).
- **Visual Highlight**: The active element is highlighted with a crisp blue border (input box / Card) or a subtle semi-transparent blue background fill (Card Previews), with keyboard input routed accordingly. Moving the mouse over any focusable result entry — the exact Card, a Card Preview, a Chinese preview row/button, or the Bing row — updates the same focus state as keyboard navigation.

### 1.8 Card Preview Jump-to-Exact Selection
- **Selecting a Preview runs an exact lookup**: Activating a Card Preview — by **mouse click**, or by pressing **`Enter`/`Space`** while it is the keyboard-focused item — replaces the query with an exact-only lookup of that preview's word (equivalent to typing `! <word>`, note the leading `!` followed by a space). The fuzzy result set collapses to the single exact match for that word.
- **Auto-select the resulting Card**: Immediately after such a jump, focus moves **directly onto the resulting exact-match Card** (`index 1`), so the user lands on the full definition without an extra keystroke — the Card is highlighted as selected the moment it renders.
- **Scope — Preview jumps only**: This jump-and-auto-select behavior is exclusive to Card Preview activation. Typing an exact query (`! word`) by hand instead keeps focus on the **input box** (`index 0`) so the user can keep editing. The auto-select is single-shot: any subsequent manual keystroke re-focuses the input box as usual.

### 1.9 Chinese → English Search (`word_cn`)
- **Language auto-detection**: When the input contains any CJK character, the overlay switches to Chinese → English mode and queries a **separate** Chinese dictionary provider (`word_cn`). Latin input keeps the existing English → Chinese flow.
- **Fuzzy term matching**: The typed Chinese is matched (prefix + edit-distance, same ranking as English) against the Chinese headword list to produce **up to 5** matching terms. There is no exact "Card" in this mode — every match is a preview row.
- **Preview row layout**: Each of the (≤ 5) rows shows the **Chinese term on the left** and **up to 3 English word buttons on the right**, ordered by English usage frequency (the underlying record stores up to 10; the previews surface the top 3).
- **Two-level keyboard focus**:
  - **Up/Down** move between the preview rows (selecting a whole row), and also exit button selection — pressing Up/Down while on an English button leaves that row for the adjacent one. The final Down reaches the always-present "Search on Bing" entry.
  - **Right** enters the focused row's English buttons (cancelling the whole-row selection) and then advances rightward through them.
  - **Left** moves back through the buttons; from the first button it returns to whole-row selection.
  - **`Enter`/`Space`** on a focused English button — or a **mouse click** on any button — runs an exact lookup of that English word and switches to the English → Chinese Card view (reusing the jump-to-exact behavior in 1.8).
- **Visual highlight**: The selected row uses a semi-transparent blue fill; the focused English button gets a blue border + fill, mirroring the English-mode highlighting.
