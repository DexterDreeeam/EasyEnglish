# Core::storage Sub-Module — Test Specification v2 (iter-015-core-storage)

Tests for this sub-module live in `Core/tests/test_storage*.rs` (per Rust
integration-test convention). Sub-module-internal helpers may live in
`#[cfg(test)] mod tests` inside `src/storage/*.rs` where the unit under test
is private.

This file is the contract — every entry below maps to one `#[test] fn`;
every `#[test] fn` in the eventual code maps back here.

## `test_storage_query_fuzzy.rs`

- `score_exact_is_one_point_zero` — verify a query exactly matching a known headword yields `score == 1.0`.
- `score_prefix_is_zero_point_nine_five` — `query("app")` against `apple` → 0.95.
- `score_levenshtein_is_below_zero_nine_five` — `query("aple")` against `apple` → score < 0.95 and > 0.
- `score_case_insensitive` — `query("APPLE")` against `apple` → 1.0.
- `query_ordering_is_score_desc_within_source` — `query("app")` against `apple` / `apply` / `book` → first two come before `book`, and either `apple` or `apply` first (both prefix).
- `query_yields_records_with_distinct_matched_words` — within one source, each matched_word appears at most once.
- `query_excludes_records_below_min_score` — `QueryOptions { min_score: 0.5, .. }` drops anything below.
- `query_yields_score_in_zero_one_inclusive` — every record has score in `[0.0, 1.0]`.
- `query_is_exact_consistent_with_score` — for every record, `is_exact == (score >= 1.0 - f32::EPSILON)`.
