## `test_storage_signals.rs`

(These exercise `Signal` / `SignalReader` directly, without spinning up a
full `Storage`. They live in the integration test crate because `Signal`
is on `Storage`'s public surface.)

- `signal_new_count_is_zero`.
- `signal_reader_clones_share_underlying_count` — clone a reader, bump signal, both clones observe new count.
- `signal_bump_advances_count_by_one`.
- `signal_count_is_monotonic_under_concurrent_bumps` — 8 threads bump 1000 times each; final count == 8000; no decreasing reads observed during the run.
- `signal_bump_once_returns_true_first_time_false_after`.
- `signal_bump_once_threadsafe_only_one_returns_true` — 16 threads race `bump_once`; exactly one returns true; signal count is exactly 1.
- `signal_wait_changed_returns_immediately_if_already_above` — bump first, then `wait_changed(0, None)`; returns 1 instantly (no block).
- `signal_wait_changed_blocks_until_bump` — main thread blocks `wait_changed`; spawn thread bumps after 50ms; main returns within ~50ms with new count.
- `signal_wait_changed_returns_last_seen_on_timeout` — `wait_changed(5, Some(50ms))` while count == 5 → returns 5 within ~50ms.
- `signal_wait_changed_wakes_on_cancel_via_completed_signal` — paired with `QueryResult` integration; cross-listed.
- `signal_wait_any_returns_first_signal_changed` — three signals; bump signal 1 mid-wait; `wait_any` returns `Some(1)`.
- `signal_wait_any_returns_first_when_multiple_change_simultaneously` — bump signal 0 and signal 2; `wait_any` returns `Some(0)` (priority by slice order).
- `signal_wait_any_returns_none_on_timeout`.
- `signal_wait_any_empty_slice_returns_none_on_timeout` — `wait_any(&[], &[], Some(10ms))` → `None`.
- `signal_wait_any_zero_timeout_returns_immediately_if_no_change`.
- `signal_reader_is_clone_send_sync` — compile-time bound test.
- `signal_is_not_clone` — compile-fail test gated by `#[cfg(...)]` ensures `let _: ()  = ::core::clone::Clone::clone::<Signal>;` doesn't compile (uses `compile_error!` via `trybuild` or a doc-test).

## `test_storage_query_growth.rs` (with `updated_signal` additions)

All existing v3 tests, plus:

- `result_updated_signal_initial_count_is_zero`.
- `result_updated_signal_bumps_on_each_append` — fixture with 5 candidates; observe `updated_signal().count()` ≥ 5 by completion.
- `result_updated_signal_count_matches_len_at_completion` — after `wait_for(usize::MAX, Some(2s))`, `updated_signal().count() as usize == result.len()`.
- `result_updated_signal_does_not_bump_after_completed_signal` — record `completed.count` at first bump; `updated.count()` is stable after that point.
- `result_updated_signal_unblocks_wait_changed` — main blocks on `updated_signal().wait_changed(0, None)`; worker pushes; main returns.
- `result_wait_for_observes_updated_signal` — verify by `bump_once` cannot wake `wait_for` (it must use the actual `updated_signal`, not a private signal).
- `result_updated_signal_independent_clones_observe_same_count`.

## `test_storage_query_finish.rs` (with `completed_signal` additions)

All existing v3 tests, plus:

- `result_completed_signal_initial_count_is_zero`.
- `result_completed_signal_bumps_once_on_natural_completion` — completed count == 1 after the worker finishes.
- `result_completed_signal_bumps_once_on_cancel` — cancel mid-run; completed count == 1.
- `result_completed_signal_does_not_bump_twice_even_if_cancel_then_finish` — call `cancel()` then let worker drain; completed count remains 1.
- `result_completed_signal_does_not_bump_twice_even_with_repeated_cancels` — call `cancel()` 5 times; completed count == 1.
- `result_completed_signal_unblocks_concurrent_waiters` — three threads each wait_changed on completed; cancel from main; all three return within 100ms.
- `result_drop_bumps_completed_signal_for_arc_clones` — `Arc::new(result)`, clone; drop original; clone observes completed count == 1.

## `test_storage_local_only.rs`

These guard against accidentally letting the network in.

- `dict_origin_has_only_seeded_json_and_sqlite_variants` — exhaustive `match` over `DictOrigin`; passing this test means no `Online` variant exists.
- `storage_config_serializes_with_no_online_variant_in_schema` — round-trip serde-json schema check.
- `ee_core_cargo_toml_has_no_http_client_dep` — read `Cargo.toml` at test runtime; assert dep set is disjoint from `["reqwest", "ureq", "hyper", "isahc", "tokio"]`. Test bound to ee-core's own Cargo.toml via `env!("CARGO_MANIFEST_DIR")`.
- `record_kind_does_not_include_online`.
- `storage_new_with_only_local_sources_succeeds` — sanity: nothing here forces a network call.
