# Core::storage Sub-Module — Interface (v4)

> **Status:** **draft (proposal v4, iter-015-core-storage).** Supersedes
> v1/v2/v3. Frozen at the end of iter-015-core-storage; breaking changes
> thereafter require an ADR.

`Storage`'s public surface is still **exactly two functions** (constructor
+ query). v4 adds two `SignalReader` accessors on `QueryResult`, plus the
supporting `Signal` / `SignalReader` types. Storage is **strictly local** —
no online dictionary integration here.

---

## Public

### `Storage`

```rust
pub struct Storage { /* opaque */ }

impl Storage {
    /// Construct: open user db + every dictionary listed in `config`.
    /// Returns:
    /// - `Err(EmptyConfig)` if `config.dictionaries.is_empty()`.
    /// - `Err(DictIdConflict(id))` if two `DictSource`s share an id.
    /// - `Err(SeedRead | SeedParse)` if a `SeededJson` origin can't load.
    /// - `Err(Sqlite | Io | Migration | Corruption)` on file-level errors.
    pub fn new(config: StorageConfig) -> Result<Self, StorageError>;

    /// Submit `term` for fuzzy search across every loaded LOCAL source.
    /// Returns immediately. A background worker thread appends `Record`s
    /// to the returned `QueryResult`'s internal list in score-desc order
    /// and bumps the appropriate signals as it goes.
    ///
    /// `term` is trimmed. Empty/whitespace term returns an already-finished
    /// `QueryResult` with `len() == 0`.
    pub fn query(&self, term: &str) -> QueryResult;
}
```

### `QueryResult` (the mutable result list + two signals)

```rust
pub struct QueryResult { /* opaque: Arc<Inner> */ }   // Send + Sync

impl QueryResult {
    // ---- v3 polling surface (unchanged) -------------------------------
    pub fn cancel(&self);
    pub fn is_finished(&self) -> bool;
    pub fn was_cancelled(&self) -> bool;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;
    pub fn get(&self, index: usize) -> Option<Record>;
    pub fn snapshot(&self) -> Vec<Record>;
    pub fn wait_for(&self, min_len: usize, timeout: Option<Duration>) -> usize;

    // ---- v4 signal surface --------------------------------------------
    /// Counter that bumps once per appended `Record`. Initial 0. Terminal
    /// value = `len()` at completion. Never bumps after `completed_signal`.
    pub fn updated_signal(&self) -> SignalReader;

    /// Counter that bumps exactly once at terminal state (cancel OR
    /// natural finish). Initial 0; terminal 1. Idempotent.
    pub fn completed_signal(&self) -> SignalReader;
}

impl Drop for QueryResult {
    /// RAII cancel. Also bumps `completed_signal` exactly once.
    fn drop(&mut self) { self.cancel(); }
}
```

### `Signal` + `SignalReader`

```rust
/// Monotonic counter + condvar. The producer side. Carry one of these on
/// the worker thread; expose `reader()` to UI.
pub struct Signal { /* opaque: Arc<SignalInner> */ }

impl Signal {
    pub fn new() -> Self;

    /// Current counter. O(1) atomic load. May increment between calls.
    pub fn count(&self) -> u64;

    /// Increment counter by 1 and wake every waiter. Idempotency is the
    /// caller's responsibility (see `Signal::bump_once`).
    pub fn bump(&self);

    /// `bump()` only if not previously called. Returns whether the bump
    /// actually fired. Threadsafe.
    pub fn bump_once(&self) -> bool;

    /// Block the calling thread until `count() != last_seen` OR `timeout`
    /// elapses. Returns the count at wake-up time (equals `last_seen` on
    /// pure timeout). `None` timeout = wait forever.
    pub fn wait_changed(&self, last_seen: u64, timeout: Option<Duration>) -> u64;

    /// Read-only handle. Cheap clone (Arc bump).
    pub fn reader(&self) -> SignalReader;

    /// Free function: wait until ANY of `signals` differs from the matching
    /// `last_seen`. Returns the index of the first changed signal, or
    /// `None` on timeout. `signals.len() == last_seen.len()` is required
    /// (debug-assert).
    pub fn wait_any(
        signals:   &[&SignalReader],
        last_seen: &[u64],
        timeout:   Option<Duration>,
    ) -> Option<usize>;
}

#[derive(Clone)]
pub struct SignalReader { /* opaque: Arc<SignalInner> */ }

impl SignalReader {
    pub fn count(&self) -> u64;
    pub fn wait_changed(&self, last_seen: u64, timeout: Option<Duration>) -> u64;
}
```

`Signal` and `SignalReader` are both `Send + Sync`. `SignalReader: Clone`.
`Signal: !Clone` (deliberate — sharing producer power is dangerous; use
`reader()`).

### Records (unchanged from v3 — no online variant)

    pub fn is_exact(&self)     -> bool;

    pub fn kind(&self)         -> RecordKind;

    pub fn as_note(&self)    -> Option<&NoteRecord>;

### Configuration (local only)

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct StorageConfig {
    pub user_db_path: Option<PathBuf>,        // None ⇒ in-memory
    pub dictionaries: Vec<DictSource>,        // ≥ 1
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct DictSource {
    pub id:           String,
    pub display_name: String,
    pub origin:       DictOrigin,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub enum DictOrigin {
    /// On first open: read json_path, write SQLite to cache_path (None ⇒
    /// in-memory cache, rebuilt each `Storage::new`). Subsequent opens
    /// attach cache and verify `seed_hash` matches the JSON.
    SeededJson { json_path: PathBuf, cache_path: Option<PathBuf> },

    /// Existing local SQLite file. Read-only.
    Sqlite { path: PathBuf },
}
```

**There is no `Online { … }` variant. This is intentional and permanent for
this module.**

### `StorageError`

```rust
#[derive(thiserror::Error, Debug)]
pub enum StorageError {
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),

    #[error("sqlite error: {0}")]
    Sqlite(#[from] rusqlite::Error),

    #[error("seed file read error: {0}")]
    SeedRead(std::io::Error),

    #[error("seed file parse error: {0}")]
    SeedParse(#[from] serde_json::Error),

    #[error("duplicate dict id: '{0}'")]
    DictIdConflict(String),

    #[error("schema migration on source '{source}' from v{from} to v{to} failed: {reason}")]
    Migration { source: String, from: u32, to: u32, reason: String },

    #[error("storage corruption: {0}")]
    Corruption(String),

    #[error("StorageConfig.dictionaries is empty")]
    EmptyConfig,
}
```

No network-related variants — by design.

## Private (not re-exported)

- `src/storage/inner.rs` — `Inner { records: Mutex<Vec<Record>>, cancelled:
  AtomicBool, finished: AtomicBool, updated: Signal, completed: Signal }`.
- `src/storage/signal.rs` — `SignalInner { count: AtomicU64, mu: Mutex<()>,
  cv: Condvar }`; `Signal` and `SignalReader` wrap `Arc<SignalInner>`.
- `src/storage/engine.rs` — k-way merge over per-source iterators.
- `src/storage/scoring.rs` — `fn score(q: &str, w: &str) -> f32` per design §9.
- `src/storage/dict.rs` / `user.rs` — per-file open / cursor helpers.
- `src/storage/schema.rs` + `migrations.rs` — per-file versioning.
- `src/storage/seed.rs` — JSON parse + sha256 hashing.
- `src/storage/conn.rs` — applies project PRAGMAs (WAL, busy_timeout, …).

## Dependencies

Workspace deps used:

- `rusqlite = { ..., features = ["bundled"] }`
- `serde` + `serde_json`
- `thiserror`
- `sha2` (for `seed_hash` cache validation)
- `std::thread`, `std::sync::{Arc, Mutex, Condvar}`, `std::sync::atomic::{AtomicBool, AtomicU64}`

Explicitly NOT used:

- No `tokio`, no `async-std`, no `futures`.
- No `reqwest`, no `ureq`, no `hyper`, no `isahc`, no HTTP client.
- No `tracing` in iter-015 scope.
- No `directories` (config provides paths).

---

## Forbidden

- Any third `pub fn` on `Storage`. The surface is **exactly two**:
  `Storage::new` and `Storage::query`.
- `pub fn` exposing `&Connection`, `MutexGuard`, `JoinHandle`, or any other
  internal type.
- `Storage::execute_sql(...)` or any raw-SQL escape hatch.
- `async fn` anywhere.
- `Storage::in_memory` as a separate constructor.
- Writes (`notes_set` etc.) on `Storage`.
- Iterator-yielding accessors on `Storage` beyond `query`.
- **Any HTTP / network IO inside `ee-core`.** Storage is local; online
  dict integration belongs to Skill/Agent if it ever ships.
- A `pub` cloneable producer-side `Signal`. Sharing the producer is hard
  to reason about; sharing the reader is what UIs need.

---

## Compatibility with iter-014 frozen surface

Same migration plan as v3 §12 — `Storage` retires `NoteStore` /
`HistoryStore` / `LookupService` / `LookupHit` / `LookupError`. ADR-0005
lands with iter-015 implementation. The added signals don't change the
migration plan; they make event-driven UIs feasible without polling, which
the platform crates will appreciate.
