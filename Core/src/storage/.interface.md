# Core::storage Sub-Module — Interface (v3)

> **Status:** **draft (proposal v3, iter-015-core-storage).** Supersedes v1
> and v2. Frozen at the end of iter-015-core-storage; breaking changes
> thereafter require an ADR.

The crate's public surface from this sub-module is, by user mandate,
exactly two functions on `Storage`:

```rust
impl Storage {
    pub fn new(config: StorageConfig) -> Result<Self, StorageError>;
    pub fn query(&self, term: &str) -> QueryResult;
}
```

Everything else below is either supporting data (config / records / errors)
or the read-only accessors on `QueryResult`.

---

## Public

### `Storage`

```rust
pub struct Storage { /* opaque */ }

impl Storage {
    /// Construct: open the user db and every dictionary listed in `config`.
    /// Each dict source is attached as its own SQLite connection; the user
    /// db is opened (or created) at `config.user_db_path` (None ⇒ in-memory).
    /// Forward schema migrations run automatically per file.
    ///
    /// Blocks until every source is ready. Returns:
    /// - `Err(EmptyConfig)` if `config.dictionaries.is_empty()`.
    /// - `Err(DictIdConflict(id))` if two `DictSource`s share an id.
    /// - `Err(SeedRead | SeedParse)` if a `SeededJson` origin can't be loaded.
    /// - `Err(Sqlite | Io | Migration | Corruption)` on file-level failures.
    pub fn new(config: StorageConfig) -> Result<Self, StorageError>;

    /// Submit `term` for fuzzy search across every loaded source. Returns
    /// immediately with a `QueryResult` whose internal list grows on a
    /// background worker thread as records are produced.
    ///
    /// `term` is trimmed before use. An empty/whitespace-only `term` returns
    /// a `QueryResult` that is already finished with `len() == 0`.
    pub fn query(&self, term: &str) -> QueryResult;
}
```

That is the entire `Storage` surface. Two methods.

### `QueryResult` — the mutable result list

```rust
pub struct QueryResult { /* opaque: Arc<Inner> */ }

impl QueryResult {
    /// Signal the background worker to stop at the next yield boundary.
    /// Idempotent. Already-produced records remain accessible.
    pub fn cancel(&self);

    /// `true` once either `cancel()` was called OR the worker ran to
    /// natural completion. Does not distinguish the two — see `was_cancelled`.
    pub fn is_finished(&self) -> bool;

    /// `true` only if the finish was due to `cancel()`. `false` for natural
    /// completion. `false` while still running.
    pub fn was_cancelled(&self) -> bool;

    /// Current number of records. Monotonically non-decreasing.
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;

    /// Clone of the record at `index`. None if out of bounds.
    /// O(1) under a short Mutex lock.
    pub fn get(&self, index: usize) -> Option<Record>;

    /// Clone of every record currently in the list, in their current order
    /// (score-desc). O(n).
    pub fn snapshot(&self) -> Vec<Record>;

    /// Block the current thread until either `len() >= min_len` or the
    /// query finishes, whichever first. Returns the `len()` at wakeup.
    /// `timeout = None` waits indefinitely. Wakes immediately on cancel.
    pub fn wait_for(&self, min_len: usize, timeout: Option<Duration>) -> usize;
}

impl Drop for QueryResult {
    /// RAII cancel. Equivalent to calling `cancel()` explicitly.
    fn drop(&mut self) { self.cancel(); }
}
```

`QueryResult: Send + Sync` (the internal `Arc<Inner>` is). Sharing via
`Arc<QueryResult>` lets one thread own iteration while another thread
calls `cancel()`.

### `Record` and friends

```rust
#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub enum Record {
    Dict(DictRecord),
    Note(NoteRecord),
    History(HistoryRecord),
}

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct BaseRecord {
    pub query_term:   String,
    pub matched_word: String,
    pub is_exact:     bool,
    pub score:        f32,
    pub kind:         RecordKind,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
pub enum RecordKind { Dict, Note, History }

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct DictRecord    { pub base: BaseRecord, pub dict_id: String, pub entry: Entry }

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct NoteRecord    { pub base: BaseRecord, pub note: Note }

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct HistoryRecord { pub base: BaseRecord, pub history: HistoryEntry }

impl Record {
    pub fn base(&self) -> &BaseRecord;

    pub fn query_term(&self)   -> &str;
    pub fn matched_word(&self) -> &str;
    pub fn is_exact(&self)     -> bool;
    pub fn score(&self)        -> f32;
    pub fn kind(&self)         -> RecordKind;

    pub fn as_dict(&self)    -> Option<&DictRecord>;
    pub fn as_note(&self)    -> Option<&NoteRecord>;
    pub fn as_history(&self) -> Option<&HistoryRecord>;
}
```

### Raw row types

```rust

    pub updated_at: i64,

    pub word:        String,
    pub recorded_at: i64,

```

Note: `Note.created_at` / `updated_at` and `HistoryEntry.recorded_at` are
populated by whoever wrote those rows (out of scope for this sub-module —
see design §10 q.1). `Storage` reads them as-is.

### `StorageConfig`

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct StorageConfig {
    /// `None` ⇒ in-memory user db (tests / transient sessions).
    pub user_db_path: Option<PathBuf>,

    /// Non-empty by validation. Order = priority (tie-breaker only).
    pub dictionaries: Vec<DictSource>,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct DictSource {
    pub id:           String,        // kebab-case, unique within config
    pub display_name: String,
    pub origin:       DictOrigin,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub enum DictOrigin {
    /// On first open: read json_path, write SQLite to cache_path
    /// (None ⇒ in-memory cache, rebuilt each `Storage::new`).
    /// On subsequent opens: attach cache and verify seed_hash matches.
    SeededJson {
        json_path:  PathBuf,
        cache_path: Option<PathBuf>,
    },

    /// Open an existing read-only SQLite file.
    Sqlite { path: PathBuf },
}
```

### `StorageError`

```rust
#[derive(thiserror::Error, Debug)]
pub enum StorageError {
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),

    #[error("sqlite error: {0}")]
    Sqlite(#[from] rusqlite::Error),

    #[error("seed file read error: {0}")]
    SeedRead(std::io::Error),

    #[error("seed file parse error: {0}")]
    SeedParse(#[from] serde_json::Error),

    #[error("duplicate dict id: '{0}'")]
    DictIdConflict(String),

    #[error("schema migration on source '{source}' from v{from} to v{to} failed: {reason}")]
    Migration { source: String, from: u32, to: u32, reason: String },

    #[error("storage corruption: {0}")]
    Corruption(String),

    #[error("StorageConfig.dictionaries is empty")]
    EmptyConfig,
}
```

Note: there is intentionally NO `InvalidInput` variant. `Storage::query`
silently treats whitespace-only `term` as empty (returns an immediately-
finished `QueryResult`).

## Private (not re-exported)

- `src/storage/inner.rs` — `Inner { records, cancelled, finished, condvar }`
  shared between `QueryResult` and the worker thread via `Arc`.
- `src/storage/engine.rs` — k-way merge over per-source iterators.
- `src/storage/scoring.rs` — `fn score(q: &str, w: &str) -> f32` (design §7).
- `src/storage/dict.rs` / `user.rs` — per-file open / cursor helpers.
- `src/storage/schema.rs` + `migrations.rs` — per-file versioning.
- `src/storage/seed.rs` — JSON parse + sha256 hashing for `seed_hash`.
- `src/storage/conn.rs` — applies project PRAGMAs (WAL, busy_timeout, …).

---

## Dependencies

Workspace deps:

- `rusqlite = { ..., features = ["bundled"] }`
- `serde` + `serde_json`
- `thiserror`
- `sha2` (for `seed_hash` in `SeededJson` cache validation)
- `std::thread`, `std::sync::{Arc, Mutex, Condvar, atomic::AtomicBool}`

No `tokio`, no `directories`, no `tracing` in iter-015 scope.

---

## Forbidden

- Any third `pub fn` on `Storage`. The surface is **exactly two**:
  `Storage::new` and `Storage::query`.
- `pub fn` exposing `&Connection`, `MutexGuard`, `JoinHandle`, or any other
  internal type.
- `Storage::execute_sql(...)` or any raw-SQL escape hatch.
- `async fn` anywhere in this sub-module.
- `Storage::in_memory` as a separate constructor (use
  `StorageConfig.user_db_path = None`).
- Writes (`notes_set`, `history_record`, etc.) on `Storage`. Writes happen
  in a separate module owned by a separate iter — see `.design.md` §10 q.1.
- Any iterator-yielding accessor on `Storage` beyond `query` (no
  `notes_iter` / `history_iter` / `dict_meta` / counts).

## Compatibility with iter-014 frozen surface

iter-014 froze `NoteStore`, `HistoryStore`, `LookupService`, `LookupHit`,
`LookupError`, and `AppState` on Core's public surface. v3 retires the
first five outright; `AppState` rewires to call `Storage::query` instead
of `LookupService::query`. Migration plan (3 iters):

| iter | scope |
|---|---|
| **iter-015-core-storage** | Implement `Storage` per this design. `NoteStore` / `HistoryStore` / `LookupService` retained as `#[deprecated]` thin facades for one release; their methods translate to `Storage` calls so existing call sites keep compiling. |
| **iter-016-retire-ee-dict** | `ee-dict` crate retired; its seed JSON moves to `Core/data/`. `Storage` becomes the only owner of dictionary persistence. |
| **iter-017-drop-deprecated** | Deprecated facades removed. Caller code uses `Storage` and `Record` directly. Re-freeze `Core/.interface.md`. ADR-0005 logs the full migration. |
