# Core::storage Sub-Module — Interface (v2)

> **Status:** **draft (proposal v2, iter-015-core-storage).** Supersedes v1.
> Frozen at the end of iter-015-core-storage. Breaking changes thereafter
> require an ADR.
>
> Re-exported as `ee_core::storage::*`; high-traffic names (`Storage`,
> `Record`, `BaseRecord`, `QueryHandle`, `CancelToken`, `StorageError`,
> `StorageConfig`) also at `ee_core::*`.

---

## Public

### `Storage`

The single owner of all persistent state. `Send + Sync`. Cheap to share via
`Arc`.

```rust
pub struct Storage { /* opaque */ }

impl Storage {
    // ---- Lifecycle ----------------------------------------------------

    /// Open every source listed in `config`. Each dict source is either
    /// opened from its cache file or seeded from its JSON if the cache
    /// doesn't exist. The user db is created if missing. Forward schema
    /// migrations run automatically; refusal modes per `.design.md` §11.
    ///
    /// Returns `EmptyConfig` if `config.dictionaries.is_empty()`.
    /// Returns `DictIdConflict(id)` if two sources share an id.
    pub fn open(config: StorageConfig) -> Result<Self, StorageError>;

    /// Open with in-memory user db AND in-memory dicts. Each dict source
    /// is still seeded; `cache_path` is ignored. Equivalent of `open` for
    /// tests.
    pub fn in_memory(config: StorageConfig) -> Result<Self, StorageError>;

    pub fn config(&self) -> &StorageConfig;
    pub fn schema_versions(&self) -> SchemaVersions;

    // ---- Source inspection (cheap, synchronous) -----------------------

    /// Ids of every loaded dict, in load (priority) order.
    pub fn dict_ids(&self) -> Vec<String>;

    /// Owned snapshot of a dict's metadata. None if `id` unknown.
    pub fn dict_meta(&self, id: &str) -> Option<DictMeta>;

    pub fn dict_entry_count(&self, id: &str) -> Result<usize, StorageError>;
    pub fn notes_count(&self)               -> Result<usize, StorageError>;
    pub fn history_count(&self)             -> Result<usize, StorageError>;

    // ---- THE query (fuzzy, streaming, cancellable) --------------------

    /// Run a fuzzy query across every enabled source. Returns immediately
    /// with a `QueryHandle`; no work has happened yet. Work is driven by
    /// `QueryHandle::next()`. Calling `cancel()` on the handle (or its
    /// `cancel_token`), or dropping the handle, stops all backing work at
    /// the next yield boundary.
    ///
    /// `term` is trimmed before use. An empty trimmed `term` returns a
    /// handle whose iterator immediately yields None.
    pub fn query(&self, term: &str, options: QueryOptions) -> QueryHandle;

    // ---- Mutations (writes; NOT queries — exact targeting is fine) ----

    /// Insert or update a note. Returns whether a new row was created or
    /// an existing one updated.
    /// Returns `InvalidInput` if `word.trim().is_empty()`.
    /// Empty `content` is allowed (and means "favorite-only" semantics).
    pub fn notes_set(&self, word: &str, content: &str) -> Result<NotesSetOutcome, StorageError>;

    /// Returns `true` if a row was deleted, `false` if no such note.
    /// Idempotent.
    pub fn notes_remove(&self, word: &str) -> Result<bool, StorageError>;

    /// Record a successful lookup the user committed to (e.g. clicked).
    /// `source_kind` describes which kind of `Record` the user accepted;
    /// `source_ref` is the `DictSource.id` for `Dict`, `None` for `Note`.
    /// `max_entries == 0` is a no-op (caller disabled history).
    /// Dedup is move-to-front (case-insensitive on `word`).
    pub fn history_record(
        &self,
        word:        &str,
        source_kind: SourceKind,
        source_ref:  Option<&str>,
        max_entries: usize,
    ) -> Result<(), StorageError>;

    pub fn history_clear(&self) -> Result<(), StorageError>;

    // ---- Enumeration (management views; NOT search) -------------------

    pub fn notes_iter(&self)   -> NotesIter;     // alpha order
    pub fn history_iter(&self) -> HistoryIter;   // most-recent-first
}
```

### `StorageConfig`

```rust
#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct StorageConfig {
    /// `None` → in-memory user db (tests, transient sessions).
    pub user_db_path: Option<PathBuf>,

    /// Non-empty by validation. Order is priority (first = highest).
    pub dictionaries: Vec<DictSource>,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub struct DictSource {
    /// kebab-case, unique within `StorageConfig.dictionaries`. Examples:
    /// "ee-builtin", "cc-cedict", "user-glossary".
    pub id:           String,

    /// Human-readable name shown in UI / "manage dictionaries" screen.
    pub display_name: String,

    pub origin:       DictOrigin,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]
pub enum DictOrigin {
    /// On first open: read json_path, write SQLite to cache_path, attach.
    /// On subsequent opens: attach cache_path (verifying the seed hash in
    /// the `meta` table matches json_path).
    SeededJson {
        json_path:  PathBuf,
        cache_path: PathBuf,
    },

    /// Open an existing SQLite file. Read-only.
    Sqlite { path: PathBuf },
}
```

### Records

```rust
/// The base attributes every record carries. Read directly via accessors
/// on `Record`, or matched on directly via `Record::*::base`.
#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct BaseRecord {
    pub query_term:   String,    // what the user typed (after trim)
    pub matched_word: String,    // canonical word in the source
    pub is_exact:     bool,      // score >= 1.0
    pub score:        f32,       // 0.0..=1.0
    pub kind:         RecordKind,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
pub enum RecordKind { Dict, Note, History }

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub enum Record {
    Dict(DictRecord),
    Note(NoteRecord),
    History(HistoryRecord),
}

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct DictRecord {
    pub base:    BaseRecord,
    pub dict_id: String,     // which DictSource produced this
    pub entry:   Entry,
}

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct NoteRecord {
    pub base: BaseRecord,
    pub note: Note,
}

#[derive(Debug, Clone, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct HistoryRecord {
    pub base:    BaseRecord,
    pub history: HistoryEntry,
}

impl Record {
    pub fn base(&self) -> &BaseRecord;

    // Pass-through accessors for the common base fields:
    pub fn query_term(&self)   -> &str;
    pub fn matched_word(&self) -> &str;
    pub fn is_exact(&self)     -> bool;
    pub fn score(&self)        -> f32;
    pub fn kind(&self)         -> RecordKind;

    // "Downcast" accessors — return Some only when the variant matches.
    pub fn as_dict(&self)    -> Option<&DictRecord>;
    pub fn as_note(&self)    -> Option<&NoteRecord>;
    pub fn as_history(&self) -> Option<&HistoryRecord>;

    pub fn into_dict(self)    -> Result<DictRecord,    Record>;
    pub fn into_note(self)    -> Result<NoteRecord,    Record>;
    pub fn into_history(self) -> Result<HistoryRecord, Record>;
}
```

### `QueryOptions`, `FirstYieldPolicy`

```rust
#[derive(Debug, Clone)]
pub struct QueryOptions {
    pub include_dict:        bool,
    pub include_notes:       bool,
    pub include_history:     bool,
    pub dict_ids:            Option<Vec<String>>,  // None = all
    pub min_score:           f32,                  // [0.0, 1.0]
    pub max_per_source:      Option<usize>,        // None = unlimited
    pub first_yield_policy:  FirstYieldPolicy,
}

impl Default for QueryOptions {
    /// All sources included, no dict_id filter, min_score = 0.0,
    /// max_per_source = Some(50), policy = BlockUntilExactOrFirst.
    fn default() -> Self;
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum FirstYieldPolicy {
    /// First `next()` blocks until either the exact-match record is found
    /// by some source, or every requested source has produced at least
    /// one candidate, whichever comes first. Subsequent yields are
    /// score-desc as produced.
    BlockUntilExactOrFirst,
    /// Yield records in score-desc order as soon as they are produced.
    YieldAsFound,
}
```

### `QueryHandle`, `CancelToken`, `QueryStats`

```rust
pub struct QueryHandle { /* opaque */ }   // Send, !Sync

impl QueryHandle {
    /// Idempotent. After calling, subsequent `next()` returns None and
    /// `is_cancelled()` returns true.
    pub fn cancel(&self);
    pub fn is_cancelled(&self) -> bool;
    pub fn query_term(&self) -> &str;

    /// A cheap clone-able cancel-only handle. Use this to cancel from a
    /// different thread than the consumer.
    pub fn cancel_token(&self) -> CancelToken;

    /// Cumulative stats up to now. Cheap snapshot.
    pub fn stats(&self) -> QueryStats;
}

impl Iterator for QueryHandle {
    type Item = Record;
    fn next(&mut self) -> Option<Record>;
}

impl Drop for QueryHandle {
    fn drop(&mut self) { self.cancel(); }  // RAII cancel
}

#[derive(Clone)]
pub struct CancelToken { /* opaque: Arc<AtomicBool> */ }  // Send + Sync + Clone

impl CancelToken {
    pub fn cancel(&self);
    pub fn is_cancelled(&self) -> bool;
}

#[derive(Debug, Clone)]
pub struct QueryStats {
    pub yielded_total:    usize,
    pub yielded_per_source: Vec<(String, usize)>,   // dict_ids + "notes" + "history"
    pub dropped_min_score: usize,
    pub errors_per_source: Vec<(String, String)>,    // String to avoid borrow into Storage
}
```

### `StorageError`

```rust
#[derive(thiserror::Error, Debug)]
pub enum StorageError {
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),

    #[error("sqlite error: {0}")]
    Sqlite(#[from] rusqlite::Error),

    #[error("seed file read error: {0}")]
    SeedRead(std::io::Error),

    #[error("seed file parse error: {0}")]
    SeedParse(#[from] serde_json::Error),

    #[error("duplicate dict id: '{0}'")]
    DictIdConflict(String),

    #[error("unknown dict id: '{0}'")]
    DictIdNotFound(String),

    #[error("schema migration on source '{source}' from v{from} to v{to} failed: {reason}")]
    Migration { source: String, from: u32, to: u32, reason: String },

    #[error("storage corruption: {0}")]
    Corruption(String),

    #[error("invalid input: {0}")]
    InvalidInput(String),

    #[error("StorageConfig.dictionaries is empty")]
    EmptyConfig,
}
```

## Private (not re-exported)

- `src/storage/engine.rs` — `QueryEngine`, the k-way-merge heap that backs `QueryHandle`.
- `src/storage/scoring.rs` — `fn score(query: &str, candidate: &str) -> f32` per `.design.md` §4.5.
- `src/storage/conn.rs` — opens a Connection with project PRAGMAs.
- `src/storage/dict.rs` — per-dict open / seed / cursor helpers.
- `src/storage/user.rs` — user-db open + notes/history statement caches.
- `src/storage/schema.rs` + `migrations.rs` — versioning, per file kind.
- `src/storage/seed.rs` — seed JSON parsing + hashing.

---

## Dependencies

Workspace deps used:

- `rusqlite = { ..., features = ["bundled"] }`
- `serde` + `serde_json`
- `thiserror`
- `sha2` (for `seed_hash` in dict meta — verifies cache validity)
- `std` only besides the above. **No** `tokio`, **no** `directories`,
  **no** `tracing` in iter-015 scope.

---

## Forbidden

- `pub fn` returning `&Connection`, `&mut Connection`, or any `MutexGuard`.
- `Storage::execute_sql(...)` or any raw-SQL escape.
- `async fn` anywhere in this sub-module.
- `Storage::dict_lookup` / `Storage::notes_get` / `Storage::dict_suggest`
  — all queries flow through `Storage::query`. (Exception: `notes_remove`
  is a write, not a query.)
- Sharing a single `Connection` across files. One file = one connection.
- A `QueryHandle` that is `Sync`. One consumer per handle, by contract.

---

## Compatibility with iter-014 frozen surface

Same breaking impact as v1, but the new `Record` enum and `QueryHandle`
replace more public types. The migration plan in `.design.md` §12 keeps
`NoteStore` / `HistoryStore` as `#[deprecated]` facades for one release; the
old `LookupService::query` signature changes to return a `Record` instead
of a `LookupHit`. ADR `0005-storage-merge.md` lands with the
iter-015-core-storage implementation.
