# Core::storage Sub-Module — Interface

> **Status:** **draft (proposal, iter-015-core-storage).** Frozen at the end
> of iter-015-core-storage. Breaking changes to anything in `## Public` after
> that point require an ADR.
>
> This is a **sub-module** of the `ee-core` crate. Once shipped, the types
> below are re-exported from `ee_core::storage::*` and the most-used names
> (`Storage`, `StorageError`, `Entry`, `Note`, `HistoryEntry`) are also
> re-exported at `ee_core::*` for convenience.

## Public

### `Storage`

The single owner of all persistent state. `Send + Sync`. Cheap to clone via
`Arc`.

```rust
pub struct Storage { /* opaque */ }

impl Storage {
    // ---- Lifecycle ----------------------------------------------------

    /// Open an existing storage file. Migrates the schema forward if needed.
    /// Errors if the file does not exist, is unreadable, is corrupted, or
    /// was created by a newer binary.
    pub fn open(path: impl AsRef<Path>) -> Result<Self, StorageError>;

    /// Open or create. If the file does not exist, create it with the
    /// current schema and seed the `entries` table from `seed_json_path`.
    /// If the file exists, behaves like `open` (migrates if necessary).
    pub fn create_or_open(
        path:           impl AsRef<Path>,
        seed_json_path: impl AsRef<Path>,
    ) -> Result<Self, StorageError>;

    /// Open an in-memory SQLite (`:memory:`) with the current schema, no seed.
    /// All other methods work identically; `path()` returns `None`.
    pub fn in_memory() -> Result<Self, StorageError>;

    /// Open an in-memory SQLite, then seed `entries` from `seed_json_path`.
    /// Equivalent to `in_memory()` followed by a private seed call.
    pub fn in_memory_with_seed(
        seed_json_path: impl AsRef<Path>,
    ) -> Result<Self, StorageError>;

    /// Path of the underlying SQLite file. `None` for in-memory storages.
    pub fn path(&self) -> Option<&Path>;

    /// Current schema version (matches `MAX(version) FROM schema_version`).
    pub fn schema_version(&self) -> u32;

    // ---- Dictionary (entries table) -----------------------------------

    /// Exact, case-insensitive headword lookup. `Ok(None)` if no such entry.
    /// `InvalidInput` if `word.trim().is_empty()`.
    pub fn dict_lookup(&self, word: &str) -> Result<Option<Entry>, StorageError>;

    /// Suggest up to `max` headwords most similar to `prefix`/`word`.
    /// Ordering: exact prefix match first, then by Levenshtein distance.
    /// `Ok(vec![])` if no candidates. `max == 0` returns `Ok(vec![])`.
    pub fn dict_suggest(&self, word: &str, max: usize) -> Result<Vec<String>, StorageError>;

    pub fn dict_count(&self) -> Result<usize, StorageError>;

    // ---- Notes ---------------------------------------------------------

    /// Insert or update a note. `word` is lowercased on storage; the original
    /// case is not preserved (matches v0.x semantics).
    /// `InvalidInput` if `word.trim().is_empty()`.
    /// `created_at` stays put on update; `updated_at` always refreshes.
    pub fn notes_set(&self, word: &str, content: &str) -> Result<(), StorageError>;

    pub fn notes_get(&self, word: &str) -> Result<Option<Note>, StorageError>;

    /// Returns `true` if a row was deleted, `false` if no such note.
    /// Idempotent.
    pub fn notes_remove(&self, word: &str) -> Result<bool, StorageError>;

    /// All notes, ordered by `word ASC`.
    pub fn notes_list(&self) -> Result<Vec<Note>, StorageError>;

    pub fn notes_count(&self) -> Result<usize, StorageError>;

    // ---- History -------------------------------------------------------

    /// Record a successful lookup. If the most-recent entry has the same
    /// `word` (case-insensitive), updates its `recorded_at` instead of
    /// inserting a new row (move-to-front semantics).
    /// Then prunes oldest rows so total count <= `max_entries`.
    /// `max_entries == 0` is a no-op (caller disabled history).
    /// `InvalidInput` if `word.trim().is_empty()`.
    pub fn history_record(&self, word: &str, max_entries: usize) -> Result<(), StorageError>;

    /// Most-recent-first, up to `max`. `max == 0` returns `vec![]`.
    pub fn history_recent(&self, max: usize) -> Result<Vec<HistoryEntry>, StorageError>;

    pub fn history_clear(&self) -> Result<(), StorageError>;

    pub fn history_count(&self) -> Result<usize, StorageError>;
}
```

### `Entry` (dictionary row)

```rust
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct Entry {
    pub headword:    String,         // canonical lowercase
    pub phonetic:    String,         // may be empty
    pub definitions: Vec<String>,    // CN strings, non-empty by invariant
}
```

### `Note` (user-defined word annotation)

```rust
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct Note {
    pub word:       String,    // canonical lowercase
    pub content:    String,    // arbitrary UTF-8; not assumed to be a translation
    pub created_at: i64,       // unix epoch seconds
    pub updated_at: i64,       // unix epoch seconds; >= created_at
}
```

### `HistoryEntry`

```rust
#[derive(Debug, Clone, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub struct HistoryEntry {
    pub word:        String,   // canonical from Entry.headword OR Note.word
    pub recorded_at: i64,      // unix epoch seconds
}
```

### `StorageError`

```rust
#[derive(thiserror::Error, Debug)]
pub enum StorageError {
    #[error("io error: {0}")]
    Io(#[from] std::io::Error),

    #[error("sqlite error: {0}")]
    Sqlite(#[from] rusqlite::Error),

    #[error("seed file read error: {0}")]
    SeedRead(std::io::Error),

    #[error("seed file parse error: {0}")]
    SeedParse(#[from] serde_json::Error),

    #[error("schema migration from v{from} to v{to} failed: {reason}")]
    Migration { from: u32, to: u32, reason: String },

    #[error("storage corruption: {0}")]
    Corruption(String),

    #[error("invalid input: {0}")]
    InvalidInput(String),
}
```

> `Io` and `SeedRead` both wrap `std::io::Error` but are kept separate so the
> caller can render "couldn't open your storage" vs "couldn't read the
> bundled dictionary seed" with different recovery hints.

## Private (not re-exported)

- `src/storage/schema.rs` — `CURRENT_SCHEMA_VERSION: u32`, `SCHEMA_V1_SQL: &str`.
- `src/storage/migrations.rs` — `MIGRATIONS: &[Migration]` where
  `Migration = (u32, fn(&mut Connection) -> Result<(), rusqlite::Error>)`.
  Phase 1 contains only `(1, init_schema)`.
- `src/storage/seed.rs` — `fn seed_entries(conn: &Connection, path: &Path) -> Result<usize, StorageError>`.
- `src/storage/dict.rs` / `notes.rs` / `history.rs` — one prepared-statement
  helper per table; never `pub`.
- `src/storage/conn.rs` — opens a Connection with the project's PRAGMAs
  applied (WAL, synchronous=NORMAL, foreign_keys=ON, busy_timeout).

## Dependencies

Workspace deps used:

- `rusqlite = { ..., features = ["bundled"] }`
- `serde` + `serde_json`
- `thiserror`
- `std` only — **no** `directories`, **no** `tracing` (Core::config keeps the
  former; the latter lands when Agent module ships).

## Forbidden

- `pub fn` returning `&Connection`, `&mut Connection`, or `MutexGuard<'_, _>`.
  Callers must not see SQLite types.
- `Storage::execute(sql: &str)` or any raw-SQL escape hatch.
- `async fn` anywhere in this sub-module.
- Re-exporting `rusqlite::Error` directly — wrap in `StorageError::Sqlite`.
- A second SQLite Connection inside the same `Storage` (use one, period).

## Compatibility with iter-014 frozen surface

This sub-module **breaks** the frozen iter-014 `Core::interface.md` in two
ways:

1. `NoteStore` and `HistoryStore` cease to exist as concrete types. They are
   replaced by methods on `Storage`. Migration plan in `.design.md` §12
   preserves the names as `#[deprecated]` thin facades for one release.
2. The `LookupService::new(notes, ...)` constructor changes to accept
   `&Storage` instead of `&NoteStore`. Functionality preserved; signature
   different.

Both changes require ADR `docs/adr/0005-storage-merge.md`, which lands
**with** iter-015-core-storage, not as part of this design review.
