# Core Module ‚Äî Test Specification

Tests live in `Core/tests/` plus a handful of in-crate unit tests where the
implementation detail is too small to deserve a public surface (e.g.
`config::tests::defaults_match_documentation`, the `algo` ranking tests, and the
`generator` bin parsing tests).

All integration tests construct a fresh in-memory `DictStore` (in-memory
sqlite, no disk I/O) so test runs are fully parallel-safe and need zero
cleanup.

## Test files

| File | What it covers |
|---|---|
| `test_storage.rs` | `Storage` database operations and connection creation |
| `test_search.rs` | `Search` online service stub and RecordProvider compiler checks |
| `test_record.rs` | `RecordModel` polymorphic JSON serialization and type mappings |
| `test_hub.rs` | `Hub` streaming queries and cancellation thread controls |
| `word_list_test.rs` | Comprehensive 1,000-word overlap checks across three real databases |

## `test_storage.rs`

- `new_storage_initializes_empty_tables` ‚Äî checks that new Storage database initialized cleanly and is queryable.
- `insert_or_update_persists_string` ‚Äî checks that values are successfully written and retrieved.

## `test_search.rs`

- `search_new_panics_unimplemented` ‚Äî checks that Search is not yet instantiable.

## `test_record.rs`

- `test_word_en_serialization_and_deserialization` ‚Äî validates complete serialization loop for WordEn.
- `test_note_and_history_plain_serialization` ‚Äî validates Note and History flat serialization.

## `test_hub.rs`

- `hub_queries_the_real_dictionary` ‚Äî loads the bundled dictionary and asserts the `apple` query returns exactly one fully-populated `WordEn` record.
- `hub_can_be_cancelled_mid_get` ‚Äî verifies that thread cancels instantaneously when canceled.

## `word_list_test.rs`

- `word_list_test` ‚Äî samples 900 real headwords (each must hit exactly once) + 100 nonexistent words (zero hits) against the single bundled dictionary, asserting hit counts and deserialization.

## In-crate unit tests

### `algo` (`src/algo/mod.rs`)
- `test_levenshtein_basic` / `test_levenshtein_unicode` ‚Äî edit-distance correctness incl. multi-byte chars.
- `test_rank_candidates` ‚Äî basic prefix + distance ranking.
- `optimized_matches_naive_reference` ‚Äî proves the length-band / bounded-distance fast path returns identical results to a naive full scan across many queries.
- `length_band_keeps_exactly_distance_three` ‚Äî boundary of the distance‚â§3 filter.
- `prefix_beyond_threshold_is_surfaced` ‚Äî prefix matches are kept regardless of edit distance and ordered by their true distance.

### `generator` (`src/bin/generator.rs`)
ECDICT ‚Üí `WordEn` import helpers (pure parsing, no DB):
- `clean_word_filters_shape` ‚Äî single ASCII-word shape filter.
- `inflection_of_other_detected` ‚Äî drops derived forms via ECDICT `0:lemma`.
- `split_pos_extracts_known_pos` ‚Äî splits a leading `n.`/`vt.` POS token.
- `parse_translation_builds_major_and_defs` ‚Äî major gloss + per-line definitions.
- `parse_inflections_maps_exchange` ‚Äî maps `exchange` `p/d/i/3/s` to `Inflections`.

## Datasets

`word_en_v1.sqlite` + `word_en_v1` (the headword list, no extension) hold the
100k-headword set imported from ECDICT by `src/bin/generator.rs` (top words by
corpus frequency with a Chinese translation; proper nouns and pure inflected
forms excluded). The ECDICT source DB (`Dict/ecdict.db`) is a gitignored build
input, so this dataset is regenerated, not unit-tested directly.
