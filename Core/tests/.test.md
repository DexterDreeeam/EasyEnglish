# Core Module ‚Äî Test Specification

Tests live in `Core/tests/` plus a handful of in-crate unit tests where the
implementation detail is too small to deserve a public surface (currently
only `config::tests::defaults_match_documentation`).

All integration tests construct a fresh in-memory `DictStore` (in-memory
sqlite, no disk I/O) so test runs are fully parallel-safe and need zero
cleanup.

## Test files

| File | What it covers |
|---|---|
| `test_config.rs` | `Config::load` happy path + missing-file error + defaults override |
| `test_notes.rs` | `NoteStore`: empty-default / set+get / overwrite / remove idempotent / case-insensitivity / list ordering |
| `test_history.rs` | `HistoryStore`: cap = 0 no-op / record / dedup-and-move-to-front / cap eviction / clear |
| `test_lookup.rs` | `LookupService`: Note-first hit / Note-first dict fallback / Dict-first hit / Dict-first note fallback / both miss / invalid input / case-insensitive |
| `test_state.rs` | `AppState`: input round-trip / empty submit is no-op / submit hit records history / submit miss clears last_hit / reset / add+remove note / get_note |

## `test_config.rs`

- `load_real_product_json_uses_documented_defaults` ‚Äî loads the workspace's
  shipped `product.json` and asserts every getter matches the documented
  default (history_max_entries == 50, notes_persist == false, etc.).
- `load_missing_file_returns_io_error` ‚Äî `Config::load` on a non-existent
  path returns `ConfigError::Io`.
- `load_malformed_json_returns_parse_error` ‚Äî `Config::load` on a file with
  garbled JSON returns `ConfigError::Parse`.
- `partial_override_keeps_other_defaults` ‚Äî a JSON that only sets
  `core.history.max_entries` keeps every other field at its default.

## `test_notes.rs`

- `new_is_empty` ‚Äî `NoteStore::new()` reports `len() == 0` and `is_empty()`.
- `set_then_get_returns_note` ‚Äî `set("apple", "fruit")` then `get("apple")`
  returns `Some(Note{word:"apple", content:"fruit"})`.
- `set_lowercases_word` ‚Äî `set("Apple", _)` is found via `get("apple")` /
  `get("APPLE")`.
- `set_overwrites_existing` ‚Äî calling `set` twice for the same word keeps
  `len() == 1` and the second content wins.
- `set_empty_is_noop` ‚Äî `set("", _)` and `set("   ", _)` leave the store empty.
- `remove_existing_returns_true` / `remove_missing_returns_false`.
- `remove_is_case_insensitive`.
- `list_is_sorted_by_word_ascending`.

## `test_history.rs`

- `cap_zero_record_is_noop` ‚Äî `HistoryStore::with_capacity(0)` ignores `record`.
- `record_single` ‚Äî records one entry; `recent()` returns it; `len() == 1`.
- `record_dedups_existing_word_and_moves_to_front` ‚Äî record `apple`, then
  `banana`, then `apple` again ‚Üí recent == `[apple, banana]`, len 2.
- `record_evicts_oldest_when_over_cap` ‚Äî cap = 2, record `a`, `b`, `c`
  ‚Üí recent == `[c, b]`.
- `record_empty_input_is_noop`.
- `clear_empties_the_store`.

## `test_lookup.rs`

Helper `mini_dict()` returns a `DictStore` seeded with three entries
(`apple` / `apply` / `book`).

- `note_first_hit_returns_note_without_touching_dict` ‚Äî set note for `apple`,
  query `apple` ‚Üí `LookupHit::Note`; the dict still has `apple` but isn't
  consulted (verified by content mismatch).
- `note_first_falls_back_to_dict_when_no_note` ‚Äî query `apply` (only in dict)
  ‚Üí `LookupHit::Dict`.
- `dict_first_hit_returns_dict_even_when_note_exists` ‚Äî construct with
  `prefer_notes_over_dict = false`, set a note for `apple`, query `apple` ‚Üí
  `LookupHit::Dict`.
- `dict_first_falls_back_to_note_when_dict_misses` ‚Äî `prefer_notes_over_dict
  = false`, no dict entry for `mango`, but a note for `mango` ‚Üí `LookupHit::Note`.
- `both_miss_returns_not_found`.
- `invalid_input_short_circuits_before_either_store`.
- `case_insensitive_in_both_orderings` ‚Äî `"APPLE"` hits whether note-first or
  dict-first.

## `test_state.rs`

- `default_state_is_empty` ‚Äî `input()` empty, `last_hit()` None, `status()`
  empty, `recent()` empty.
- `submit_empty_input_is_noop` ‚Äî `submit()` on an empty input doesn't update
  status or last_hit.
- `submit_hit_records_to_history_and_sets_status_found` ‚Äî after submit on a
  hit, `last_hit()` is Some and `recent()` contains the canonical word.
- `submit_miss_clears_last_hit_with_not_found_status`.
- `reset_clears_buffer_and_hit`.
- `add_note_then_submit_returns_note_hit` ‚Äî note takes priority over dict.
- `remove_note_then_submit_returns_dict_hit_again`.
- `get_note_round_trip`.
