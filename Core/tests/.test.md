# Core Module — Test Specification

Tests live in `Core/tests/` plus a handful of in-crate unit tests where the
implementation detail is too small to deserve a public surface (currently
only `config::tests::defaults_match_documentation`).

All integration tests construct a fresh in-memory `DictStore` (in-memory
sqlite, no disk I/O) so test runs are fully parallel-safe and need zero
cleanup.

## Test files

| File | What it covers |
|---|---|
| `test_storage.rs` | `Storage` database operations and connection creation |
| `test_search.rs` | `Search` online service stub and RecordProvider compiler checks |
| `test_record.rs` | `RecordModel` polymorphic JSON serialization and type mappings |
| `test_hub.rs` | `Hub` streaming queries and cancellation thread controls |
| `word_list_test.rs` | Comprehensive 1,000-word overlap checks across three real databases |

## `test_storage.rs`

- `new_storage_initializes_empty_tables` — checks that new Storage database initialized cleanly and is queryable.
- `insert_or_update_persists_string` — checks that values are successfully written and retrieved.

## `test_search.rs`

- `search_new_panics_unimplemented` — checks that Search is not yet instantiable.

## `test_record.rs`

- `test_word_en_serialization_and_deserialization` — validates complete serialization loop for WordEn.
- `test_note_and_history_plain_serialization` — validates Note and History flat serialization.

## `test_hub.rs`

- `hub_concurrently_queries_three_real_dbs` — queries three databases concurrently and asserts matching records.
- `hub_can_be_cancelled_mid_get` — verifies that thread cancels instantaneously when canceled.

## `word_list_test.rs`

- `word_list_test` — streams, deserializes, and asserts 1,000 overlapping/non-overlapping words across word_en_v1.sqlite, v2.sqlite, and v3.sqlite databases.

## Quality gate

```powershell
cargo nextest run -p ee-core
cargo fmt --all --check
cargo clippy --workspace --all-targets -- -D warnings
```

All three must pass before commit.
