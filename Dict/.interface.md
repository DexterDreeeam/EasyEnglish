# Dict Module — Interface

> **Status:** **frozen** (since iter-013). Any breaking change to the surface
> below requires an ADR.

## Public

### `Entry` (struct)

```rust
pub struct Entry {
    pub headword: String,           // canonical lower-case form
    pub phonetic: String,           // may be empty
    pub definitions: Vec<String>,   // Chinese senses, in display order
}
```

### `DictError` (enum)

```rust
#[non_exhaustive]
pub enum DictError {
    NotFound,
    InvalidInput,
    Storage(rusqlite::Error),
    Seed(String),
}
```

`DictError` implements `std::error::Error` via `thiserror`.

### `DictStore`

```rust
pub struct DictStore { /* opaque */ }

impl DictStore {
    /// Open an existing SQLite dictionary database. Returns `DictError::Storage`
    /// if the file is missing or unreadable.
    pub fn open(db_path: impl AsRef<Path>) -> Result<Self, DictError>;

    /// Open the database if it exists, otherwise seed it from `seed_path`
    /// (a JSON file produced by `Dict/data/seed_en_cn.json`). Idempotent on
    /// subsequent calls.
    pub fn create_or_seed(
        db_path: impl AsRef<Path>,
        seed_path: impl AsRef<Path>,
    ) -> Result<Self, DictError>;

    /// Exact case-insensitive lookup. Returns `DictError::NotFound` if the word
    /// is absent, `DictError::InvalidInput` for empty / oversized input.
    pub fn lookup(&self, word: &str) -> Result<Entry, DictError>;

    /// Suggestions ordered by ascending Levenshtein distance (alphabetical
    /// tiebreak). Empty / oversized input returns an empty vector — never errors.
    pub fn suggest(&self, prefix: &str, max: usize) -> Vec<String>;
}
```

Invariants (must be preserved by every release):

- `lookup` is case-insensitive but `Entry::headword` is always returned in the seed's
  canonical lower-case form.
- `lookup` of an empty string or a string longer than 128 bytes returns
  `DictError::InvalidInput` without hitting the database.
- `suggest` never returns an error; the worst case is an empty vector.
- `DictStore` is `Send` (callable from worker threads) but is **not** `Sync` — wrap in
  `Mutex` if shared.

## Private

The opaque inner state of `DictStore` includes a `rusqlite::Connection`, a prepared
`SELECT` statement for `lookup`, and an in-memory `Vec<String>` cache of every
lower-cased headword used by `suggest`. These are implementation details and must not
be exposed.
