# Dict Module — Test Specification

All tests are integration tests under `Dict/tests/`. They drive the public
`DictStore` API end-to-end. Tests use `tempfile::TempDir` to isolate each
run; no fixture is shared between tests.

The seed JSON `Dict/data/seed_en_cn.json` is the single source of truth and
is consumed verbatim by `test_seeder.rs`. Other tests build their own tiny
JSON inputs so they are robust against future expansions of the canonical
seed.

## Test files

| File | What it covers |
|---|---|
| `test_store.rs` | `DictStore::open` failure path; `DictStore::create_or_seed` round-trip; `lookup` hit / miss / case-insensitivity / invalid input; `suggest` ordering and bounds |
| `test_seeder.rs` | Loading the real `seed_en_cn.json` — sanity on entry count, ASCII headwords, every entry has at least one non-empty Chinese definition |

## `test_store.rs`

### Setup
A `tiny_seed.json` is generated inside the test's tempdir with three entries
(`apple` → ["苹果","苹果树"], `apply` → ["申请"], `book` → ["书","预订"]).
`DictStore::create_or_seed` builds a fresh sqlite file from that JSON.

### Cases

- `create_or_seed_loads_all_entries` — three entries land in the DB and the
  cache reports `len() == 3`.
- `create_or_seed_is_idempotent` — calling `create_or_seed` twice on the same
  path yields the same row count; the seed step is skipped the second time.
- `open_missing_file_returns_storage_error` — `DictStore::open` on a path that
  does not exist returns `DictError::Storage`.
- `lookup_hit_returns_canonical_entry` — `lookup("apple")` returns
  `Entry{ headword="apple", definitions=["苹果","苹果树"] }`.
- `lookup_is_case_insensitive` — `lookup("APPLE")` and `lookup("Apple")` both
  hit and the returned `headword` is `"apple"` (canonical lowercase).
- `lookup_unknown_word_returns_not_found` — a word not in the seed returns
  `DictError::NotFound`.
- `lookup_empty_returns_invalid_input` — `""` and `"   "` return
  `DictError::InvalidInput` (the trim happens before validation).
- `lookup_oversized_returns_invalid_input` — a 200-character string returns
  `DictError::InvalidInput` without touching sqlite.
- `suggest_empty_returns_empty` — `suggest("", 5)` is empty.
- `suggest_max_zero_returns_empty` — `suggest("appl", 0)` is empty.
- `suggest_exact_match_ranks_first` — `suggest("apple", 3).first() == Some("apple")`.
- `suggest_typo_ranks_correct_word_first` — `suggest("appel", 1).first() ==
  Some("apple")` (distance 1 to "apple", distance 2 to "apply").
- `suggest_respects_max_limit` — `suggest("z", 2).len() <= 2`.

## `test_seeder.rs`

- `real_seed_loads_cleanly` — opening `Dict/data/seed_en_cn.json` succeeds and
  `len() >= 200`.
- `every_entry_has_nonempty_definitions` — iterates every seed headword via
  `lookup` and asserts the returned `Entry::definitions` is non-empty.
- `headwords_are_ascii_lowercase` — every cached headword is ASCII and
  already lower-cased (no surprise unicode in the seed).
