# Skill Module — Test Specification (iter-015)

Tests will live in `Skill/tests/` once iter-015 ships. This file is the
contract — no test is added without an entry here, no entry stays without a
matching test.

All tests construct skills as `Box<dyn Skill>` and exercise them through the
trait only. Concrete builtin types are never named in test code.

## Test files

| File | Covers |
|---|---|
| `test_registry.rs` | `SkillRegistry`: empty / register / duplicate-id / get / list / ids / with_builtins shape |
| `test_skill_trait.rs` | Trait contract via an in-test `FakeSkill` — every variant of `SkillError`, no-panic guarantee |
| `test_annotation.rs` | `Annotation::new` clamping + serde round-trip + every `AnnotationKind` variant |
| `test_builtins_stub.rs` | Every shipped built-in id is present, has a description, and currently returns `DataSourceUnavailable` |

## `test_registry.rs`

- `new_registry_is_empty` — `SkillRegistry::new()` has `len() == 0`, `is_empty()`, `list()` empty, `ids()` empty.
- `register_unique_id_succeeds` — register a `FakeSkill::new("a.b")` → `Ok(())`, `len() == 1`, `contains("a.b")`.
- `register_duplicate_id_returns_duplicate_error` — register the same id twice → `Err(RegisterError::DuplicateId("a.b"))`.
- `get_known_id_returns_some` / `get_unknown_id_returns_none`.
- `list_preserves_insertion_order` — register `c`, `a`, `b` → `list()` returns `[c, a, b]`.
- `ids_returns_lexicographic_order` — same registrations → `ids()` returns `[a, b, c]`.
- `with_builtins_contains_known_ids` — every id documented in `Skill/.design.md` §2 is present (`ipa.american`, `ipa.british`, `examples.3`, `examples.5`, `etymology`, `synonyms.same-register`, `formality.note`).
- `with_builtins_has_no_duplicates` — `ids().len() == len()`.
- `with_builtins_descriptions_under_80_chars` — for every built-in `s`, `s.description().len() < 80` and contains no newline.

## `test_skill_trait.rs`

A `FakeSkill` lives in `tests/common/mod.rs`. It is parameterised by a
closure that produces its `run` result, so each test scenario is one line.

- `fake_skill_id_is_what_was_passed`.
- `fake_skill_run_returns_annotation_with_matching_source_id` — output's `source_skill_id == skill.id()`.
- `fake_skill_returns_not_applicable_when_configured` — `Err(SkillError::NotApplicable)` propagates exactly.
- `fake_skill_returns_data_source_unavailable_with_message` — message preserved verbatim.
- `fake_skill_returns_cancelled` — present so Agent can model budget hits.
- `fake_skill_returns_internal` — message preserved verbatim.
- `fake_skill_never_panics_on_empty_word` — `SkillInput{ word: "", ... }` does not panic; result is whatever the FakeSkill is configured to produce.
- `fake_skill_never_panics_on_unicode_word` — `"naïveté"` and emoji input do not panic.
- `fake_skill_never_panics_on_hit_none` — `SkillInput{ hit: None, ... }` does not panic.

## `test_annotation.rs`

- `new_clamps_confidence_below_zero_to_zero`.
- `new_clamps_confidence_above_one_to_one`.
- `new_passes_through_in_range_confidence`.
- `new_preserves_source_skill_id_and_kind_and_content`.
- `kind_ipa_round_trips_via_serde_json`.
- `kind_examples_round_trips`.
- `kind_etymology_round_trips`.
- `kind_synonyms_round_trips`.
- `kind_antonyms_round_trips`.
- `kind_formality_round_trips`.
- `kind_translation_round_trips`.
- `kind_custom_string_round_trips` — `AnnotationKind::Custom("my.kind".into())` survives JSON.
- `serialized_field_names_are_snake_case` — guards against accidental rename (`source_skill_id`, not `sourceSkillId`).

## `test_builtins_stub.rs`

For each documented built-in id:

- `<id>_is_registered_by_with_builtins`.
- `<id>_currently_returns_data_source_unavailable` — sanity that we did not accidentally implement one.
- `<id>_description_is_present_and_short`.

(These tests get replaced — *not deleted* — when a later iter implements
that builtin for real. The intent is documented in the test itself.)
