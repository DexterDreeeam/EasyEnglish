# Skill Module — Interface

> **Status:** **draft (proposal, iter-015).** Frozen at the end of iter-015.
> Once frozen, breaking changes to anything in `## Public` require an ADR.

Crate name: `ee-skill`.

## Public

### `trait Skill`

Object-safe; lives in `SkillRegistry` as `Box<dyn Skill>`.

```rust
pub trait Skill: Send + Sync {
    /// Stable identifier, e.g. "ipa.american". See `Skill/.design.md` §2 for
    /// the naming scheme. Two registrations with the same id are rejected.
    fn id(&self) -> &str;

    /// Short, human-readable description for UI / `--list-skills`.
    /// Must be < 80 chars, single line, no trailing period.
    fn description(&self) -> &str;

    /// Run the skill. Implementers must never panic; map every error to
    /// `SkillError`. Implementers must respect `input.ctx` (don't return
    /// British IPA when the caller asked for American).
    fn run(&self, input: SkillInput<'_>) -> Result<Annotation, SkillError>;
}
```

### Inputs

```rust
pub struct SkillInput<'a> {
    pub word: &'a str,                           // already trimmed by caller
    pub hit:  Option<&'a ee_core::LookupHit>,    // None if dict missed
    pub ctx:  &'a SkillContext,
}

pub struct SkillContext {
    pub region:    Region,
    pub level:     CefrLevel,
    pub formality: Formality,
}

/// Re-exported into ee-agent::Instructions; defined here so Skill is
/// self-contained. See `Skill/.design.md` §7 question 2.
pub enum Region    { US, UK, AU, IN }
pub enum CefrLevel { A1, A2, B1, B2, C1, C2 }
pub enum Formality { Casual, Neutral, Formal }
```

### Output: `Annotation`

```rust
#[derive(Clone, serde::Serialize, serde::Deserialize)]
pub struct Annotation {
    /// `Skill::id()` of the producer. The UI uses this for grouping.
    pub source_skill_id: String,
    pub kind:            AnnotationKind,
    /// UTF-8 text payload. Shape depends on `kind` — e.g. `Examples` is
    /// newline-separated; `Ipa` is a single IPA string. See per-kind doc.
    pub content:         String,
    /// 0.0..=1.0. Deterministic skills return 1.0. The crate clamps on
    /// construction; out-of-range values become 0.0 or 1.0.
    pub confidence:      f32,
}

#[derive(Clone, serde::Serialize, serde::Deserialize)]
pub enum AnnotationKind {
    Ipa,
    Examples,
    Etymology,
    Synonyms,
    Antonyms,
    Formality,
    Translation,
    Custom(String),
}

impl Annotation {
    /// Builds an Annotation; clamps confidence into [0.0, 1.0].
    pub fn new(
        source_skill_id: impl Into<String>,
        kind:            AnnotationKind,
        content:         impl Into<String>,
        confidence:      f32,
    ) -> Self;
}
```

### `SkillRegistry`

```rust
pub struct SkillRegistry { /* opaque */ }

impl SkillRegistry {
    /// Empty registry. The Agent layer typically prefers `with_builtins`.
    pub fn new() -> Self;

    /// Registry preloaded with every built-in skill stub shipped in this
    /// iter: `ipa.american`, `ipa.british`, `examples.3`, `examples.5`,
    /// `etymology`, `synonyms.same-register`, `formality.note`.
    /// All built-ins currently return `DataSourceUnavailable("...")` until
    /// a later iter adds real data.
    pub fn with_builtins() -> Self;

    /// Register a skill. Returns `Err(DuplicateId)` if `skill.id()` is
    /// already present.
    pub fn register(&mut self, skill: Box<dyn Skill>) -> Result<(), RegisterError>;

    pub fn get(&self, id: &str) -> Option<&dyn Skill>;
    pub fn contains(&self, id: &str) -> bool;
    pub fn len(&self) -> usize;
    pub fn is_empty(&self) -> bool;

    /// All registered skills in insertion order.
    pub fn list(&self) -> Vec<&dyn Skill>;

    /// All registered ids in lexicographic order — stable for UI listing.
    pub fn ids(&self) -> Vec<String>;
}

#[derive(thiserror::Error, Debug)]
pub enum RegisterError {
    #[error("a skill with id '{0}' is already registered")]
    DuplicateId(String),
}
```

## Private (not re-exported)

- `src/builtin/ipa.rs`, `src/builtin/examples.rs`, etc. — concrete builtin
  types are constructed by `SkillRegistry::with_builtins()` and remain
  hidden. Callers cannot pattern-match on them; they only see `&dyn Skill`.
- `src/registry.rs` internally is a `HashMap<String, Box<dyn Skill>>` plus a
  `Vec<String>` to preserve insertion order for `list()`.
- The clamping logic in `Annotation::new` lives in `src/annotation.rs` and
  is not exposed as a free function.

## Dependencies

Workspace deps used:

- `ee-core` (for `LookupHit`)
- `serde` + `serde_json` (for `Annotation` round-trip)
- `thiserror`
- *(no rusqlite, no httplib, no async runtime — those live in concrete
  skills that get added in later iters.)*

## Forbidden

- `pub fn` returning concrete builtin types.
- Any `impl Skill` outside this crate's `src/builtin/` (Phase 1 — once
  plugin loading is on the table this opens up).
- `async fn` on the trait.
- Panicking inside any `Skill::run` body.
