# Win Module — Design

The `Win` crate will provide the **Windows platform integration** for
EasyEnglish: tray icon, global hotkey, frameless overlay window, and the
**WiX MSI** installer pipeline.

> **Phase 1 status:** placeholder. The crate compiles as an empty library so the
> workspace layout is locked in from day one. Implementation lands in a future
> iteration (currently un-numbered — will be scheduled after `Dict` and `Core`
> are stable).

---

## Planned responsibilities

1. **Tray icon** — a system-tray-resident process that stays alive when the
   overlay is hidden. Left click shows the overlay; right click opens a
   Show / Quit menu.
2. **Global hotkey** — register a system-wide shortcut so the user can summon
   the overlay from any other application. The default will be `Alt+Space`
   for cross-platform consistency.
3. **Overlay window** — a frameless, always-on-top, single-line input widget
   that renders `ee_core::AppState` via egui (or, if a Windows-native
   toolkit makes more sense, decide in the corresponding ADR).
4. **Single instance** — refuse to start a second instance; reuse the running
   tray icon to surface the overlay instead.
5. **Focus save / restore** — capture the foreground window before showing
   the overlay; restore it when the overlay dismisses, so the user's typing
   flow is never interrupted.
6. **Packaging** — produce a WiX MSI installer driven by `cargo`
   subcommands; no GitHub Actions involved.

## Planned dependencies

windows-rs (tray + LowLevelMouseHook), winit, egui, cargo-wix

Reverse dependencies are forbidden by the architecture: this crate may depend
on `ee-core` and `ee-dict`, but neither may depend on us.
