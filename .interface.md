# EasyEnglish — Root Interface

The root of the workspace exposes no classes itself; every API surface lives inside a
submodule. This file is the single index to those.

## Public

(No types at workspace root level.)

## Submodules

| Module | Crate name | One-line interface summary |
|---|---|---|
| [`Dict`](Dict/.interface.md)   | `ee-dict`     | `Entry`, `DictError`, `DictStore` — open / seed a SQLite dictionary; `lookup` + `suggest` |
| [`Core`](Core/.interface.md)   | `ee-core`     | `Config`, `Note`, `NoteStore`, `HistoryStore`, `LookupService`, `LookupHit`, `AppState` |
| [`Utils`](Utils/.interface.md) | `ee-utils`    | `DynamicResult`, `DynamicResultProducer`, `Signal` — thread-safe growing-state with cancel/complete/fail/spawn |
| [`Win`](Win/.interface.md)     | `ee-win`      | (placeholder) future: `WindowsShell` — tray + global hotkey + overlay + MSI packaging |
| [`Mac`](Mac/.interface.md)     | `ee-mac`      | (placeholder) future: `MacShell` — tray + global hotkey + overlay + dmg packaging |
| [`Linux`](Linux/.interface.md) | `ee-linux`    | (placeholder) future: `LinuxShell` — tray + global hotkey + overlay + AppImage/deb |

## Private

`product.json` at the repo root is a *runtime configuration*, not part of the public Rust
API — `ee-core::Config` parses it. Treat the JSON shape as semi-stable; breaking changes
go in a `docs/adr/`.
