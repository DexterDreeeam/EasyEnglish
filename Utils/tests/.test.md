# Utils Module — Test Specification (iter-015)

Integration tests live in `Utils/tests/test_dr_*.rs`. In-crate unit tests
live in `#[cfg(test)] mod tests` at the bottom of `src/dynamic_result.rs`.

## Test files

| File | Covers |
|---|---|
| `test_dr_basic.rs`        | `new`, `get`, `cancel`, `update`, basic `wait`, type bounds |
| `test_dr_signals.rs`      | All four `Signal` variants, terminal-state decision table |
| `test_dr_spawn.rs`        | `spawn` natural finish, panic → `Failed`, drop semantics |
| `test_dr_producer.rs`     | `complete`, `fail`, `wait_for_cancel`, `Drop = complete` |
| `test_dr_concurrency.rs`  | Cross-thread `update`/`wait`, `Arc<DynamicResult>`, races |

In-crate unit tests in `src/dynamic_result.rs`:

- `snapshot_terminal_returns_none_initially`

- `fail_winner_freezes_others`
