# Utils Module — Design

The `Utils` module hosts reusable concurrency primitives that don't belong to
any one feature module. It is **infrastructure**, not product logic. Anything
in here must be:

- `std`-only (no third-party deps in iter-015)
- Generic / not coupled to dictionaries, lookups, UI, OS APIs
- Of unambiguous use by ≥ 2 future consumers (Core, Win, Mac, Linux)

## Current contents (iter-015)

| File | Type | Purpose |
|---|---|---|
| `dynamic_result.rs` | `DynamicResult<T>` + `DynamicResultProducer<T>` + `Signal` | Thread-safe growing-state handle with cancel + complete + fail + spawn |

See [`.interface.md`](.interface.md) for the full API contract and
[`tests/.test.md`](tests/.test.md) for the test specification.

## Rules

- **Forbidden:** any third-party dep (no tokio, no parking_lot, no crossbeam).
- **Forbidden:** `unsafe` code (`#![forbid(unsafe_code)]` at crate root).
- New utilities arrive **one per iter** with their own design + interface + test docs.
- Promotion to its own workspace member only if utility grows beyond `std`.
