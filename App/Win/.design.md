# Win Module — Windows Platform Shell

The `Win` crate (`ee-win`) provides the specialized Windows platform integrations for EasyEnglish.

All general overlay window behaviors, layout aesthetics, and interactive features conform strictly to the repository-wide [**Global Interactive UX Specification**](../.design.md). This document details only the Windows-specific native system integrations.

---

## 1. Win32 Native Integrations

To maintain a lightweight footprint and 100% compile compatibility across all platforms, `ee-win` implements native integrations using raw Windows APIs through `windows-sys` (v0.52.0) with specific GDI features.

### 1.1 Low-Level Mouse & Keyboard Hooks (`WH_MOUSE_LL` & `WH_KEYBOARD_LL`)
- **Hotkey Wakeup**: To intercept `LeftAlt + LeftShift + MouseWheelUp` and `LeftAlt + LeftShift + UpArrow` system-wide, both a global low-level mouse hook (`WH_MOUSE_LL`) and a low-level keyboard hook (`WH_KEYBOARD_LL`) are installed.
- **Modifiers Detection**: Physical keys are queried at the moment of the scroll or keydown event using `GetAsyncKeyState(VK_LSHIFT)` and `GetAsyncKeyState(VK_LMENU)`.
- **Winit Loop Suspension Bypass**: Since Windows suspends event loops of invisible windows, both hook procedures bypass winit completely. They use `FindWindowW` to locate the window by its title (`"flyout"`) and directly invoke:
  - `ShowWindow(hwnd, SW_SHOW)`
  - `SetForegroundWindow(hwnd)`
  - `ctx.request_repaint()` (to resume egui drawing)

### 1.2 System Tray Icon Daemon
- **Hidden Window**: Registers a custom Win32 window class (`WNDCLASSW`) and creates a hidden window handle to process notification tray messages.
- **Tray Registration**: Inserts a tray icon using `Shell_NotifyIconW(NIM_ADD, &nid)`.
- **Context Popup**: Upon receiving `WM_RBUTTONUP` (right click on the icon), `GetCursorPos` retrieves mouse coordinates and `TrackPopupMenu` spawns a native context menu with:
  - **Show Flyout**: Invokes native `ShowWindow`/`SetForegroundWindow` commands to wake up the search bar.
  - **Exit**: Posts a quit message, triggers global atomic exit flags, and safely cleans up all tray and hook resources.

## 2. Windows Crate Layout

The Windows-specific implementation is split into focused, single-responsibility
modules (all gated behind `#[cfg(target_os = "windows")]` via `lib.rs`):

- `src/main.rs`: Executable entrypoint that launches the background daemon on Windows.
- `src/lib.rs`: Library entrypoint containing `#[cfg(target_os = "windows")]` gates to ensure cross-compilation on non-Windows hosts is completely green; declares the private modules below and re-exports `run`.
- `src/runner.rs`: `run()` entrypoint — spawns the tray thread and starts the `eframe` app.
- `src/overlay.rs`: The `eframe`/`egui` search overlay (`SearchOverlayApp`) — window lifecycle, layout, query dispatch, and rendering.
- `src/focus.rs`: Flyout animation state (`AnimationState`) and the pure `evaluate_focus_hide` auto-hide decision (unit-tested).
- `src/signals.rs`: Cross-thread coordination statics (wake / exit / egui context), shared window handles, and `request_flyout_wakeup`.
- `src/win32.rs`: Low-level Win32 helpers — window lookup, screen metrics, and show/foreground.
- `src/tray.rs`: System tray icon, global hotkey registration, the tray window procedure, and the Win32 message loop.
- `src/dict.rs`: Dictionary database and word-list discovery (highest-version selection).
- `src/logging.rs`: Debug-only timestamped file logging sink.
