# Win Module — Design Specification & Interactions

The `Win` crate (`ee-win`) provides the specialized Windows desktop shell integrations for EasyEnglish. It implements a seamless, highly reactive, floating, zero-footprint translation flyout.

---

## 1. Core Responsibilities & Completed Architecture

The module owns the system-wide overlay trigger and floating presentation, structured cleanly to separate OS-native hooks and tray message loops from the egui rendering frame:

```
App/Win/
├── src/
│   ├── main.rs         # Background daemon executable entrypoint
│   ├── lib.rs          # Cross-platform compilation gateway & exports
│   └── imp.rs          # Windows-specific system-tray, hooks, and overlay UI
```

---

## 2. Interactive Experience & UX Specification (交互体验设计规范)

This submodule is designed around an ambient HUD experience, conforming strictly to the following refined UX rules:

### 2.1 Pure Bordered Input Box (极简黑色矩形框)
- **Visuals**: No window background, no title bars, and no blue button or magnifying glass icon.
- **Input Field**: Replaced by a high-contrast, pure deep black (`#0F0F0F`) rounded rectangle (`rounding(4.0)`) with a thin, sharp gray border (`Stroke::new(1.5, gray)`).
- **Sizing**: Compact and focused, locked at a fixed width of `460px`.

### 2.2 System-Tray Resident Daemon (系统托盘常驻后台)
- **Background Startup**: Starts silently and invisible in the background with `visible: false` window properties.
- **Notification Tray**: Registers a standard Windows notify icon.
- **Context Menu**: Right-clicking the tray icon spawns a lightweight, native popup context menu containing:
  - **`Show Flyout`**: Explicitly wakes up and displays the search box.
  - **`Exit`**: Fully terminates the message loop, uninstalls hooks, deletes notification icons, and cleanly exits the process.

### 2.3 Global Mouse-Wheel Trigger Hook (鼠标滚轮全局热键唤醒)
- **Trigger**: System-wide global `Ctrl + Shift + MouseWheelUp` (鼠标滚轮向上滑轮) combination intercepts scroll up actions globally.
- **Low-level Hook (`WH_MOUSE_LL`)**: Intercepts events instantly and retrieves keyboard states using `GetAsyncKeyState(VK_CONTROL)` and `GetAsyncKeyState(VK_SHIFT)`.
- **Winit Suspension Bypass (原生唤醒)**: Since Windows suspends invisible window event loops, the hook thread calls Win32 native **`ShowWindow`** and **`SetForegroundWindow`** by looking up the window title `"flyout"` using `FindWindowW`. This forces the window to become visible and receive OS focus instantly.

### 2.4 Focus Grace Period & Auto-Close (失焦/Esc 自动折收隐藏)
- **Dismiss on Esc**: Pressing the `Escape` key immediately clears the text buffer, clears previous queries, and hides the viewport.
- **Dismiss on Focus Loss (失焦自闭)**: Once the window loses active foreground focus (such as when clicking on any other application or on the desktop), it automatically clears text and hides itself instantly.
- **DPI-Aware Focus Grace Period (15帧焦点缓冲锁)**: Waking up a window involves asynchronous OS animations where the window is temporarily "unfocused". To prevent instant self-closing on wakeup, a **15-frame grace period** is set upon wakeup. Loss-of-focus detection is active only after the grace frames elapse, ensuring smooth and robust focus acquisition.

### 2.5 High-DPI Aware Vertical Center Auto-Resizing (全自适应屏幕居中)
- **Dynamic Sizing**: The window's height dynamically adapts in real-time based on content (starting at `56px` for a plain input box, and scaling up to `356px` to wrap query results).
- **DPI Scaling Alignment**: System metrics retrieved via `GetSystemMetrics` are scaled down using `ctx.pixels_per_point()` to align with egui's logical point coordinate system.
- **Symmetric Centering**: The window's absolute position on the screen is dynamically repositioned every frame:
  - `x = (logical_screen_width - 460.0) / 2.0`
  - `y = (logical_screen_height - desired_height) / 2.0`
  - This keeps the floating flyout perfectly centered on the screen horizontally and vertically throughout all transition states!

## 3. Native Dependencies

This module relies on:
- **`eframe`** & **`egui`** (v0.28.1): Viewport commands, transparent color clearing, and proportional font scaling.
- **`windows-sys`** (v0.52.0): With `Win32_Foundation`, `Win32_UI_WindowsAndMessaging` (including GDI), `Win32_UI_Shell`, `Win32_UI_Input_KeyboardAndMouse`, and `Win32_System_LibraryLoader`.
