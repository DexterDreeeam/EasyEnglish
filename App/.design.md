# App Module — Design

`App/` is the **platform-integration umbrella**. It is a directory grouping,
not a crate of its own. Three sibling crates live under it, one per
supported desktop OS:

| Crate | Crate name | Owns |
|---|---|---|
| [`Win`](Win/.design.md) | `ee-win` | Windows: tray icon, global hotkey, frameless overlay window, WiX MSI installer |
| [`Mac`](Mac/.design.md) | `ee-mac` | macOS: tray, global hotkey, overlay, dmg / pkg installer |
| [`Linux`](Linux/.design.md) | `ee-linux` | Linux: StatusNotifierItem tray, overlay, AppImage / deb |

Each crate is built only on its target OS (via `cfg(target_os = ...)` gating
where needed). All three depend on `ee-core` for `AppState`, on `ee-utils`
for `DynamicResult` (used to drive UI from background workers), and on
`ee-dict` indirectly through Core.

## Status

All three are **placeholders in Phase 1**. Iter-018+ will fill them in.
Documentation for individual crates lives in their respective folders.

## Why a grouping?

- Visually separates platform crates from the cross-platform layer
  (Core / Dict / Utils stay at the repo root).
- Keeps `cargo build --workspace` working unchanged — every crate is still
  a normal workspace member, just under a deeper path.
- The grouping has no Cargo.toml of its own and no public Rust surface;
  see [`.interface.md`](.interface.md).
