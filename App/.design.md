# App Module — Design Specification & Global UX Constitution

`App/` is the **platform-integration umbrella** for EasyEnglish. It is a directory grouping holding three sibling crates, one per supported desktop OS:

| Crate | Crate name | Target OS | Primary Platform Integrations |
|---|---|---|---|
| [`Win`](Win/.design.md) | `ee-win` | Windows | Tray icon, native low-level mouse hooks, Win32 message-only window |
| [`Mac`](Mac/.design.md) | `ee-mac` | macOS | Status bar item, Cocoa global event monitors, NSWindow overlays |
| [`Linux`](Linux/.design.md) | `ee-linux` | Linux | AppIndicator/StatusNotifierItem tray, X11/Wayland input overlays |

All platform crates are bound to a unified, shared **Interactive UX Constitution** to guarantee identical user behavior across all desktop environments.

---

## 1. Global Interactive UX Specification (全平台通用交互设计规范)

To maintain a consistent, distraction-free, and highly responsive user experience across Windows, macOS, and Linux, every platform shell implementation MUST conform strictly to the following architectural design:

### 1.1 Pure Bordered Input Box (极简黑色矩形框)
- **Compact Floating HUD**: There is no window decoration, title bar, close/minimize buttons, or outer canvas background. Only the search box itself is rendered as a standalone floating container.
- **Search Bar Layout**: An elegant, solid deep black (`#0F0F0F`) rectangular block with slight rounding (`rounding(4.0)`), enclosed by a crisp gray border (`stroke(1.5)`), sized strictly to a compact width of `460px`.
- **No Magnifying Glass**: Clean input interface without magnifying glass icons, search buttons, or indicators on the right. Typing is completely silent and distraction-free.

### 1.2 System-Tray Resident Daemon (系统托盘常驻后台)
- **Silent Background Startup**: On execution, the application starts completely hidden with no main window.
- **Persistent Tray Icon**: Registers an OS-native status indicator in the system tray / menu bar.
- **Tray Menu**: Right-clicking the tray icon reveals a native popup menu containing:
  - **`Show Flyout`**: Wakes up and focuses the floating search box.
  - **`Exit`**: Stops all background loops, deletes status icons, uninstalls hooks, and cleanly terminates the process.

### 1.3 Global Hot-Trigger (全局组合热键唤醒)
- **Universal Shortcut**: Pressing **`LeftAlt + LeftShift + MouseWheelUp`** (左Alt + 左Shift + 鼠标向上滚动) or **`LeftAlt + LeftShift + UpArrow`** (左Alt + 左Shift + 键盘上箭头) globally instantly triggers the search overlay.
- **Platform Hook**: The system-wide low-level mouse hook intercepts scroll-up events, checks key modifier states, and forces the hidden window to become visible and receive active keyboard focus.
- **Bypass Event Loop Suspension**: Each platform MUST implement native window showing commands (such as Win32 `ShowWindow` or Cocoa `makeKeyAndOrderFront`) directly from the hook thread to wake up suspended window event loops instantly.

### 1.4 Focus Grace Period & Auto-Close (失焦/Esc 自动折收隐藏)
- **Escape to Hide**: Pressing the `Escape` key immediately clears the text input buffer, clears search results, and hides the search window back into the tray.
- **Hide on Focus Loss**: Clicking anywhere outside the search window (losing foreground OS focus) immediately clears state and hides the window back into the tray.
- **DPI/OS Animation Grace Period (15帧焦点缓冲锁)**: Because window show animations take time (making the window temporarily unfocused during first frames), the app enforces a **15-frame grace period** on wakeup. Loss-of-focus check is active only after the grace frames elapse, preventing immediate self-closing on wakeup.

### 1.5 High-DPI Aware Vertical Center Auto-Resizing (全自适应屏幕居中)
- **Dynamic Sizing**: The window's height dynamically adapts in real-time based on content (starting at `56px` for a plain input box, and scaling up to `356px` to wrap query results).
- **DPI Scaling Alignment**: System metrics retrieved from the OS are scaled down using `ctx.pixels_per_point()` to align with egui's logical point coordinate system.
- **Symmetric Centering**: The window's absolute position on the screen is dynamically repositioned every frame:
  - `x = (logical_screen_width - 460.0) / 2.0`
  - `y = (logical_screen_height - desired_height) / 2.0`
  - This keeps the floating flyout perfectly centered on the screen horizontally and vertically throughout all transition states!
